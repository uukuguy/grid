## What landed

- `JwtClaims` gained a required `tenant_id` field (D-05: no serde default → pre-v3.8 tokens fail `validate_jwt`)
- New `mint_jwt(tenant_id, user_id, email, role, ttl_secs)` symmetric to `validate_jwt`
- `MIN_JWT_SECRET_BYTES = 32` constant (HS256 minimum per RFC 7518 §3.2)
- `try_from_env()` production constructor: refuses to construct when `mode = full` and secret is missing/short, returns Err with operator-actionable message
- `AuthMode::Full` middleware in `grid-server` inserts BOTH the existing `UserContext` (preserving the API surface) AND the new `JwtClaims` (so 03.8.2 handlers can read tenant_id for cross-tenant scoping)
- 9 hermetic tests + 3 middleware integration tests cover REQUIREMENTS.md AUTH-01, AUTH-04, AUTH-05
