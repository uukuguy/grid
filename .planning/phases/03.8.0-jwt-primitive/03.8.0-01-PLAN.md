### Task 2: Add `mint_jwt` symmetric to `validate_jwt`

**File:** `crates/grid-engine/src/auth/config.rs`

```rust
/// Issue a JWT for the given claims.
///
/// Returns the encoded token (compact JWS) and the expiration timestamp.
pub fn mint_jwt(
    &self,
    tenant_id: &str,
    user_id: &str,
    email: &str,
    role: &str,
    ttl_secs: i64,
) -> Result<(String, i64), String> {
    let secret = self
        .jwt_secret
        .as_ref()
        .ok_or_else(|| "GRID_JWT_SECRET not configured".to_string())?;

    let now = Utc::now().timestamp();
    let exp = now + ttl_secs;
    let claims = JwtClaims {
        sub: user_id.into(),
        email: email.into(),
        role: role.into(),
        tenant_id: tenant_id.into(),
        iat: now,
        exp,
    };

    let token = encode(
        &Header::new(Algorithm::HS256),
        &claims,
        &EncodingKey::from_secret(secret.as_bytes()),
    )
    .map_err(|e| format!("jwt_encode_failed: {e}"))?;

    Ok((token, exp))
}
```

- Add `encode`, `EncodingKey`, and `Header` to the `use jsonwebtoken::{...}` line at the top
- Error type = `String` (matches the existing `validate_jwt` style which returns `Option<JwtClaims>`); richer error enum defers to 03.8.2 when RBAC + cross-tenant rejections introduce more failure modes

### Task 3: Lock down `GRID_JWT_SECRET` env loading

**File:** `crates/grid-server/src/config.rs` (or `crates/grid-engine/src/auth/config.rs::Default`, TBD by file path grep)

- In `AuthConfig::default()` (or its constructor used by `grid-server` config loader):
  - Read `GRID_JWT_SECRET` env var
  - If `mode = full` and `GRID_JWT_SECRET` is unset/empty â†’ log WARN + store as `None` (the mint/verify path then returns errors with actionable messages; do NOT panic at config load â€” panic only at mint time, when the operator actually tries to issue a token without a secret)
