## What landed

- **`TenantContext::for_multi_user(tenant_id, user_id, role)`** constructor — engine-side, no leg-specific branches (per ADR-V2-023 P1).
- **`require_action_middleware`** now has two enforcement paths: JWT path (`Extension<JwtClaims>` → `Role::parse` → `TenantContext::for_multi_user` → `Role::can(action)`) + legacy `UserContext::has_permission` path (preserved for AuthMode::None / ApiKey).
- **Tenant-scoped `SessionStore` accessors** with default implementations: `get_session_for_tenant` returns the new 3-armed `TenantSessionResult` enum (`Ok | TenantMismatch | NotFound`); `list_sessions_for_tenant` filters enumeration to caller's scope.
- **AUDIT-02 cross_tenant escape hatch** on `GET /audit?cross_tenant=true`: every such request is recorded as a SECURITY row tagged with the caller's user_id + tenant_id + role; non-Owner callers receive 403.
- **9 hermetic RBAC/tenant tests** in `crates/grid-server/tests/multi_user_rbac_tenant.rs`.

## Out of scope (carried to v3.9+)

- DB-backed `users` table (D-01 of 03.8.1 plan)
