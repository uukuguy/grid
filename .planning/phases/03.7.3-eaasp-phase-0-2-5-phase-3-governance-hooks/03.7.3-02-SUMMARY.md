### Persistence contract (audit §6, implemented)

`governance_decisions` table (9 columns + 2 CHECK constraints + 1 index):

```sql
CREATE TABLE IF NOT EXISTS governance_decisions (
    decision_id TEXT PRIMARY KEY,
    session_id  TEXT NOT NULL,
    hook_id     TEXT NOT NULL,
    tool_name   TEXT NOT NULL,
    risk_level  TEXT NOT NULL CHECK(risk_level IN ('read','write_local','write_external')),
    decision    TEXT NOT NULL CHECK(decision IN ('allow','approve','deny','gate_request')),
    approver    TEXT,
    rationale   TEXT NOT NULL,
    ts          TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_governance_decisions_session_ts
    ON governance_decisions(session_id, ts DESC);
```

Append-only semantics: request (`gd_<uuid>`) and final
(`gd_<uuid>_final`) decisions are separate rows with distinct primary
keys. Final decisions NEVER overwrite request rows.

### L4 governance event payloads (audit §7.1, implemented)

`governance.request` payload — exactly:
```json
{"decision_id": "gd_xxx", "hook_id": "h_pre", "tool_name": "scada_set_setpoint",
 "risk_level": "write_external", "action_preview": "xfmr-042/temperature_limit_c=70.0"}
```

`governance.decision` payload — exactly:
```json
{"decision_id": "gd_xxx_final", "decision": "approve", "approver": "cli:--yes"}
```

Best-effort failure: when `append()` raises, helper logs loguru warning
and returns `None`; the L3 audit ledger remains authoritative.

### CLI approval UX (audit §7.2, implemented)

- `--yes` and `--no` flags mutually exclusive (exit 2 if both)
- `--yes` → `("approve", "cli:--yes")`; `--no` → `("deny", "cli:--no")`
- No flags → `typer.confirm("Approve?", default=False)` → interactive
- Default deny (empty / `n`) → `("deny", "cli:interactive")`
- Deny → `typer.Exit(4)` (no tool dispatch)
