## DO-NOT-TOUCH: Session Fixture

`tests/contract/conftest.py:113-124` — `@pytest.fixture(scope="session") def runtime_name(request)`. This is the **legitimate session-scoped fixture** that reads `--runtime=<X>`. Bug is at the parametrize layer shadowing it; fixture itself is correct.

```python
@pytest.fixture(scope="session")
def runtime_name(request: pytest.FixtureRequest) -> str:
    value = request.config.getoption("--runtime")
    if value is None:
        pytest.skip("--runtime not supplied; contract_v1 requires --runtime=<name>")
    return value
```

Downstream session-scoped `runtime_config` fixture (conftest.py:241) depends on `runtime_name` — this dependency chain is why pytest raised `ScopeMismatch` when parametrize injected a function-scoped name into the same slot.

## Edit-Tool Pattern Recommendation

For each test file: chain of `Edit` calls (one per string swap above) is cleaner than sed-style multi-line because:
1. The Edit tool enforces read-before-edit (matches CLAUDE.md rule)
2. Each unique f-string at unique line — no risk of multi-match collision
3. Planner can list each Edit invocation 1:1 with the table rows above

**Exception:** decorator + function-sig pair (e.g. L139+L142) can batch as one multi-line `old_string` if planner prefers fewer tool calls — both lines are 3 LOC apart and uniquely-identifiable.
