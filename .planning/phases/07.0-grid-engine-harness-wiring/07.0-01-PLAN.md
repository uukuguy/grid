    ### Sub-item B: Make trim/compaction budget-driven

    Per CONTEXT.md D-02b: "`trim_for_budget()` becomes budget-driven (triggered by budget threshold) instead of turn-boundary driven."

    The current harness proactive compaction at `crates/grid-engine/src/agent/harness.rs:966-1018` runs when `usage_pct >= compaction_cfg.proactive_threshold_pct` **regardless of remaining budget**. The change: add a budget gate — skip compaction when there's plenty of budget remaining.

    **Change in harness.rs, inside the proactive compaction block (around L966-970):**

    Current code:
    ```rust
    if !compaction_cfg.reactive_only && !proactive_compact_done_this_turn {
        if usage_pct >= compaction_cfg.proactive_threshold_pct {
            // ... run compaction ...
    ```

    Add `task_budget_remaining < MIN_TURN_BUDGET` as a second gate:
    ```rust
    // D3 (ENGINE-02): Budget-driven compaction — skip proactive
    // compaction when we have enough token budget remaining.
    // Only compact when budget is genuinely tight (less than
    // MIN_TURN_BUDGET tokens left, meaning the next turn may not
    // fit). This avoids unnecessary LLM compaction calls on
    // every turn boundary.
    let budget_tight = task_budget_remaining < MIN_TURN_BUDGET;

    if !compaction_cfg.reactive_only
        && !proactive_compact_done_this_turn
    {
        if budget_tight && usage_pct >= compaction_cfg.proactive_threshold_pct {
            // ... existing compaction logic ...
    ```

    **Explanation:** `task_budget_remaining` is initialized at L422 as `context_window * MAX_TURNS_FOR_BUDGET` (usually ~200K * 50 = 10M tokens). It decrements after each LLM turn. When it's still high, we have budget headroom — compaction is unnecessary CPU/LLM cost. Only when the remaining budget falls below `MIN_TURN_BUDGET` (4096 tokens) do we consider the budget "tight" enough to justify the cost of compaction.

    **Add debug log when budget is NOT tight but threshold would trigger:**
    ```rust
    if budget_tight && usage_pct >= compaction_cfg.proactive_threshold_pct {
        // ... existing log + compaction ...
    } else if usage_pct >= compaction_cfg.proactive_threshold_pct {
        debug!(
            threshold = compaction_cfg.proactive_threshold_pct,
            usage_pct,
            task_budget_remaining,
            "Proactive compaction skipped — sufficient budget headroom"
        );
    }
    ```
