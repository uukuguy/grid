## Deviations

1. **`#[serde(deny_unknown_fields)]` widened to `#[serde(deny_unknown_fields, default)]`** at struct level on `CompactionPipelineConfig` in T02. The plan's T01 step 1 specified bare `deny_unknown_fields`, but T02's `missing_compaction_block_uses_default` test exposed that a PARTIAL `compaction:` block (e.g. only `proactive_threshold_pct: 60`) required ALL 10 fields. The struct-level `default` attribute makes each missing field fall back to the field's `Default::default()` value while preserving the strict-rejection of unknown keys. This is the canonical serde pattern; the plan's stricter wording would have produced an unusable YAML surface (users would need to specify all 10 fields every time). The behavior matches ADR-V2-028 intent: tolerate omissions, reject typos.

2. **Test name `initialize_injects_memory_refs_as_system_preamble`** in `harness_payload_integration.rs` (no `test_` prefix), not `test_initialize_injects_memory_refs_preamble` as the plan text said. Treated as a scrivener slip; used the actual function name.

3. **D58 Send-path observation strategy: pure shared-helper, not session_store read-back.** The plan's Step 3 sketched assertions via `harness.runtime().session_store().get_session(...).await`, but the executor only persists `initial_history` to `session_store` after a real LLM turn (see `crates/grid-engine/src/agent/executor.rs:579-583` — `store.set_messages(...)` fires inside `AgentMessage::UserMessage` handler, never at session start). The integration test has no live provider. The fallback option the plan authorized — "add a test-only accessor" — is implemented as the new `pub fn build_initial_history_for_payload`, called identically from both `initialize()` and the test, so the test's snapshot is bit-equal to the engine's input. The Send-path coverage intent is satisfied (the test now proves the preamble actually lands in the `Vec<ChatMessage>` shipped into the executor) without standing up a mock LLM.

4. **`make config-gen` regenerates `config.yaml`, NOT `config.default.yaml`.** The plan's Step 9 mentioned `config.default.yaml`. In this repo, the Makefile target `config-gen` writes `config.yaml`, and the tracked `config.default.yaml` is a stale snapshot (last touched when crates were named `octo-*`, references "Octo Configuration"). The canonical source for default YAML rendering is `Config::generate_default_yaml()` in `crates/grid-server/src/config.rs`, which now includes a `compaction:` section documenting all 10 fields with their defaults. `config.yaml` was not staged (would mix in unrelated noise from older changes); `config.default.yaml` is left alone (would need a milestone-level refresh pass to update the "Octo" branding too). The new compaction section is fully documented in source.

5. **No Cargo.lock change.** Plan T01 step 3 hedged "if absent under `[dev-dependencies]`, add `serde_yaml = "0.9"`". `serde_yaml` was already in `[dependencies]` of both grid-engine and grid-server (via workspace deps), so no Cargo.toml edit was needed and Cargo.lock is unchanged.
