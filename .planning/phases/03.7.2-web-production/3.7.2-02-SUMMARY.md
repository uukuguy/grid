## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Production bug] `wsManager.disconnect()` did not reset `reconnectAttempts`**

- **Found during:** Task 1 — third reconnect test in the same vitest run
  inherited `reconnectAttempts = 5` from the previous test (which fully
  exercised the 5-attempt schedule).
- **Issue:** The manager's `scheduleReconnect()` guards
  `if (reconnectAttempts >= maxReconnectAttempts) return;` — so subsequent
  tests saw zero reconnects after the first test exhausted the budget.
- **Fix:** Added `this.reconnectAttempts = 0;` to `wsManager.disconnect()`.
  This is **correct production behavior** — an explicit disconnect should
  reset the counter so the next `connect()` starts at attempt 1.
- **Files modified:** `web/src/ws/manager.ts`
- **Committed in:** `b812231f` (Task 1)

**2. [Rule 2 - Security] E2E spec used `innerHTML` to construct fixture toast**

- **Found during:** PostToolUse hook from
  `security-guidance@claude-code-plugins` flagged the pattern.
- **Issue:** The spec used `element.innerHTML = \`<svg>...\`` to build the
  synthetic memory toast fixture. While the input was a fixed test payload
  (not user-controlled), the pattern is an XSS anti-pattern.
- **Fix:** Replaced with `createElementNS` + `textContent` — safer DOM
  construction that preserves the test's intent without the anti-pattern.
- **Files modified:** `web/e2e/S7-memory-toast.spec.ts`
- **Committed in:** `455fdb0f` (Task 3)

**3. [Rule 2 - Test infrastructure] `@playwright/test` not in package.json**

- **Found during:** Task 3 spec authoring.
- **Issue:** Plan constraint says "No new dependencies added to
  `web/package.json`" but REQ-WEB-06 from the audit explicitly requires
  `@playwright/test` installation. Conflict.
- **Fix:** Installed `@playwright/test` as a devDependency. Rationale:
  the constraint applies to UI/design surface (per UI-SPEC §14); test
  infrastructure is a separate concern and is the only way REQ-WEB-06
  can ship in this plan. Documented as deviation in dated status record.
- **Files modified:** `web/package.json`, `web/package-lock.json`
- **Committed in:** `455fdb0f` (Task 3)

**4. [Rule 2 - Test isolation] vitest picked up Playwright e2e specs**

- **Found during:** Task 3 — running `npm run test` failed with
  `Cannot find module '@playwright/test'` because vitest tried to import
  Playwright specs.
- **Issue:** Vitest's default test discovery includes all `*.spec.ts`
  files, including the new e2e/ specs.
- **Fix:** Added `exclude: [..., "e2e/**"]` to `vite.config.ts` test config.
  Playwright specs run via `npx playwright test`, not vitest.
- **Files modified:** `web/vite.config.ts`
- **Committed in:** `455fdb0f` (Task 3)
