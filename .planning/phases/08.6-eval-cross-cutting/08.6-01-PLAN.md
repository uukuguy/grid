Current state: The wipe block (line 220-224) clears `data/verify-v2-phase2-*.db` and `*-shm`/`*-wal` files, but NOT `data/verify-v2-phase2-skill-registry/` directory used by the skill-registry Rust binary (`--data-dir` at line 304).
Fix: Add `rm -rf "$PROJECT_ROOT/data/verify-v2-phase2-skill-registry"` to the wipe block.

### EVAL-03 (D128): @assertion decorator NOTE-before-PASS ordering
Current state: The `@assertion` decorator in both `verify-v2-phase2.py` (line 88-104) and `verify-v2-mvp.py` (line 77-97) calls `fn()` first (which may print diagnostic NOTE messages from function body), THEN prints `PASS N. name`. Reading order is `NOTE: ...\nPASS 5. ...` — confusing interleaving.
Fix: In BOTH decorators, add `print(f"  NOTE {num:2d}. {name}")` BEFORE calling `fn()` to create a clear NOTE header. The order becomes: `NOTE N. name` → function body prints → `PASS N. name`.

### EVAL-04 (D129): `verify-v2-phase2.sh` cleanup trap port sweep guard
Current state: The `cleanup()` trap (line 79-95) runs on EXIT/INT/TERM. `_kill_tree` guards on PID being empty (no-op if unset), BUT `_kill_port` sweeps ports unconditionally via `lsof`. If pre-flight fails before any service starts, the trap still `kill -KILL`s any process listening on ports 18081-18085 and 50051-50052 — potentially killing services from another terminal (e.g., `make dev-eaasp`).
Fix: Wrap each `_kill_port` call in cleanup() with a guard on the corresponding PID variable being non-empty. Pattern: `[ -n "$L2_PID" ] && _kill_port "L2 memory-engine" "$L2_MEM_PORT"`. Same for all 6 services.

**In `scripts/verify-v2-phase2.py` (lines 88-104):**
Change the `wrapped()` function to print a NOTE line BEFORE calling `fn()`:

```python
def assertion(num: int, name: str) -> Callable[[Callable[[], None]], Callable[[], None]]:
    def decorate(fn: Callable[[], None]) -> Callable[[], None]:
        def wrapped() -> None:
            print(f"  NOTE {num:2d}. {name}")
            try:
                fn()
                results.append((num, name, "PASS", None))
                print(f"  PASS {num:2d}. {name}")
            except AssertionError as e:
                results.append((num, name, "FAIL", str(e)))
                print(f"  FAIL {num:2d}. {name}")
                print(f"         Reason: {e}")
            except Exception as e:  # pragma: no cover — diagnostic only
                results.append((num, name, "ERROR", repr(e)))
                print(f"  ERR  {num:2d}. {name}")
                print(f"         Error: {e!r}")
        return wrapped
    return decorate
```

Apply the identical change — add `print(f"  NOTE {num:2d}. {name}")` as the first line of `wrapped()`, before the `try:` block.

The output ordering becomes:

  [any function body diagnostic prints]
  PASS  1. D87 regression test passes...
