## Inline tech-debt fixes (scope-bleed acknowledged)

Three pre-existing studio-feature test bugs were blocking
`cargo test -p grid-cli --features studio`. Fixing them was the cheapest
way to get CI coverage of the new T-01.14/19 tests under the studio path.
Each is a 1-line fix:

1. **Wrong module path in 5 test asserts** (`vim_normal.rs:162,164,174` +
   `vim_insert.rs:238,240`). The `super::super::widgets::figures::VimMode`
   path resolves to `key_handlers::widgets::figures::VimMode` (which doesn't
   exist). Correct path is `super::super::super::widgets::figures::VimMode`
   (one more `super` to escape `mod tests` → file module → `key_handlers` →
   `tui`). Latent since commit `92b77106` (2026-05-04 key_handler split).
2. **`test_dollar_goes_to_end`** sent `KeyCode::Char('$')` without
   `KeyModifiers::SHIFT`. Real `$` keypress on US keyboard carries SHIFT
   (Shift+4); handler at `vim_normal.rs:75` matches `(SHIFT, '$')`.
3. **`test_x_deletes_character`** expected cursor=1 after deleting at
   cursor=2 from "hello". Actual vim semantics (and impl at line 81-87):
   cursor only retreats if buffer-end overshoot. Buffer becomes "helo"
   (len 4), cursor 2 < 4, cursor stays. Test corrected to assert 2.

These three fixes did NOT alter production behavior — only test
correctness. The original 5-04 split landed without anyone running
`--features studio` lib tests, hiding all three bugs.
