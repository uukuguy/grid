# Plan 07.2-03: skill-registry scope filter + L2 archive FTS isolation

## Objective

Close D11 (LEDGER L114) by moving skill-registry's scope filter from post-query in-memory loop to a single SQL WHERE clause backed by a new `access_scope` column on the `skills` table, and close D13 (LEDGER L116) by making L2 `archive()` actually hide rows from the FTS5 search path. Both are correctness fixes (not perf); both are isolated to their own subsystem (Rust skill-registry + Python L2 archive).

Strict-by-default per ADR-V2-028 lineage is not directly invoked here (no new env override surface), but the schema migration follows the established `ALTER TABLE ... ADD COLUMN` idempotent pattern from L2's own `apply_embedding_migration` (db.py:126-153) so a re-run never double-applies.

## Notes on plan sequencing

Per CONTEXT D-08 this plan has no HARD dependency on Plans 01/02. Plan 03 touches two separate codebases (Rust + Python) and two unrelated subsystems; executor may run T01-T02 (skill-registry, Rust) and T03 (L2 archive, Python) in parallel if convenient.

  ```rust
      /// `access_scope` column for rows that already existed before the
      /// migration. For each row whose access_scope IS NULL, we read the

      async fn backfill_access_scope_from_frontmatter(&self) -> Result<()> {
          // List candidates first (cheap; metadata only). For each, read
          // SKILL.md from the filesystem and update.
          let candidates: Vec<(String, String)> = self
              .db
              .call(|conn| {
                  let mut stmt = conn.prepare(
                      "SELECT id, version FROM skills WHERE access_scope IS NULL",
                  )?;
                  let rows = stmt
                      .query_map([], |row| {
                          Ok((row.get::<_, String>(0)?, row.get::<_, String>(1)?))
                      })?
                      .filter_map(|r| r.ok())
                      .collect();
                  Ok(rows)
              })
              .await
              .context("list candidates for backfill")?;

          for (id, version) in candidates {
              let skill_path = self
                  .base_dir
                  .join("skills")
                  .join(&id)
                  .join(&version)
                  .join("SKILL.md");
              if !skill_path.exists() {
                  continue;
              }
              let Ok(content) = std::fs::read_to_string(&skill_path) else {
                  continue;
              };
              let (frontmatter_yaml, _prose) = parse_skill_md(&content);
              let Ok(parsed_v2) =
                  crate::skill_parser::parse_v2_frontmatter(&frontmatter_yaml)
              else {
                  continue;
              };
              let Some(scope_value) = parsed_v2.access_scope else {
                  continue;
              };
              let id_clone = id.clone();
              let version_clone = version.clone();
              let scope_clone = scope_value.clone();
              self.db
                  .call(move |conn| {
                      conn.execute(
                          "UPDATE skills SET access_scope = ?1 \
                           WHERE id = ?2 AND version = ?3",
                          rusqlite::params![scope_clone, id_clone, version_clone],
                      )?;
                      Ok(())
                  })
                  .await
                  .context("backfill access_scope")?;
          }
          Ok(())
      }
  ```
