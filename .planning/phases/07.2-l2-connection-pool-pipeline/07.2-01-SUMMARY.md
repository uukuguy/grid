## Deviations

1. **`.env` likely sets `EAASP_EMBEDDING_PROVIDER=ollama`**. The pytest hangs on the cache test + busy_timeout test were caused by the default singleton picking up `ollama` from `.env` and timing out on local Ollama (not running). Workaround for this exec environment: prefix every pytest invocation with `EAASP_EMBEDDING_PROVIDER=mock`. Tests themselves are correct ‚Äî they pass cleanly with mock. The skip-on-degraded path in test_hybrid_index_cache also exists per plan spec. Documented as an exec-environment caveat only.

2. **Throughput test's `index.search("hello", top_k=5)` final probe completed without timing out on the same run that hung the cache test**. The difference: throughput's single search happens once at the end; the cache test does 6 searches each blocking on the `OllamaEmbedding` HTTPX call. Both behave correctly once `EAASP_EMBEDDING_PROVIDER=mock` is set.
