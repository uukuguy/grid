## What was built / changed

### T01 — HNSW tombstone counter + rebuild trigger (commit `ca13d047`)

- `tools/eaasp-l2-memory-engine/src/eaasp_l2_memory_engine/vector_index.py`:
  * NEW module-level constant `TOMBSTONE_REBUILD_THRESHOLD = 0.30`.
  * NEW `HNSWVectorIndex._deleted_count` — monotonic per-process tombstone counter. Persisted across `save()` ↔ `_try_load_sync()` via a new `deleted_count` key in `meta.json` (older files default to 0).
  * `mark_deleted` success in `add()` overwrite branch (line ~259) + `delete()` (line ~282) now bumps the counter. Idempotent — if `mark_deleted` raises (already-marked), the counter does NOT bump, avoiding double-count.
  * `add()` checks `(deleted_count / current_count) >= 0.30` BEFORE the overwrite branch and invokes `_rebuild_locked()` while holding the existing `self._write_lock`.
  * NEW private async `_rebuild_locked()`: snapshots live labels via `hnswlib.Index.get_items(labels)`, builds a fresh `hnswlib.Index` with the same params + `max_elements`, replays via `add_items`, swaps the inner `self._index` pointer, resets `_deleted_count`, persists (`save_index` + inline `meta.json` write to avoid re-acquiring the held lock).

### T03 — embed_batch concurrency (commit `ec31ed4d`)

- `tools/eaasp-l2-memory-engine/src/eaasp_l2_memory_engine/embedding/provider.py`:
  * NEW module-level `EMBED_BATCH_CONCURRENCY_MOCK_DEFAULT = 100`, `EMBED_BATCH_CONCURRENCY_OLLAMA_DEFAULT = 5`.
  * NEW `_load_embed_batch_concurrency_override()` — reads `L2_EMBED_BATCH_CONCURRENCY` env, strict-by-default per ADR-V2-028 (`ValueError` on malformed or `< 1`). Returns None when unset.

  * `OllamaEmbedding.__init__` + `MockEmbedding.__init__` both store `self._concurrency_limit = override or per-provider-default`.
  * Both `embed_batch` methods replaced with `asyncio.gather + asyncio.Semaphore(self._concurrency_limit)` pattern. Public method signature unchanged (`list[str] -> list[list[float]]`).
- Existing `test_embedding_provider.py`: 8/8 PASS in 0.04s.
