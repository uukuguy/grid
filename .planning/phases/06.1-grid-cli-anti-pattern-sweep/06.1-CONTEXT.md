## Implementation Decisions

### CLI-X3 — `--all-features` 12 错误处理

- **D-01 (CLI-X3 strategy)**: 走 **Option (a) 全修** 而非 Option (b) 矩阵窄化或 Option (c) 半修。理由:
  - 12 错误根因清晰: **Phase BA Grid 重命名遗漏** (`octo→grid` 2026-04-04, 8 crates 重命名时漏掉 `crates/grid-engine/wit/octo-hook.wit` + `OctoHookPlugin` bindings + bindgen world name) + 独立的小 bug (1 个 `_config` typo 五处级联 + 1 处缺 `use std::collections::HashMap` + 1 处 borrow-checker 需 `let mut`)
  - wasm hook code **不是 dead code** — 被 `crates/grid-engine/src/agent/runtime.rs:686-693` `WasmHookHandler::create_engine()` + `::load()` 引用,作为 `sandbox-wasm` feature 启用时的 hook plugin 加载路径
  - 修复总量 ~30 LOC 跨 3-4 文件 (mechanical rename + 3 行小修),不需 logic change,风险低
  - Option (b) 矩阵窄化是 "hack for validation" 形态 (在 Cargo.toml 或 Makefile 加排除 comment),违反 MEMORY.md `feedback_no_hack_for_validation` 精神
  - Option (c) 半修 留下 wit/ rename 在下一个 phase, NEW-X3 在 LEDGER 仍半开,徒增 v3.3 TRIAGE 负担

- **D-02 (X3 bucket 切分,3 buckets)**:
  - **Bucket A (5 errors) — Phase BA rename leftover**: 重命名 `crates/grid-engine/wit/octo-hook.wit` → `wit/grid-hook.wit`, 文件内 `package octo:hook@1.0.0` → `package grid:hook@1.0.0`, world name 与 bindings 中 `world: "grid-hook-plugin"` 对齐 (`hooks/wasm/mod.rs:25-28` 的 `bindgen!` 宏 `world:` 字段 + path 同步), bindings 类型 `OctoHookPlugin` → `GridHookPlugin` (`hooks/wasm/handler.rs:121,127`), `octo::hook::host::Host` 路径 → `grid::hook::host::Host` (`hooks/wasm/host_impl.rs:49`)
  - **Bucket B (6 errors) — 简单 Rust bug**: 添加 `use std::collections::HashMap;` 到 `hooks/declarative/bridge.rs` (修 E0412 + 1 个 E0433); 修复 `crates/grid-engine/src/sandbox/wasm.rs:334+` 函数签名 `_config` → `config` (修 5 个 E0425 cascade,均来自同一处变量遮蔽 — 函数签名带下划线但函数体引用无下划线版本)
  - **Bucket C (1 error) — borrow-checker**: `crates/grid-engine/src/hooks/declarative/bridge.rs` 中 `bridge` 变量需声明为 `mut` 或重构调用点 (E0596 提示位置确切)

- **D-03 (X3 verify command)**: 修复完成后,**任务退出条件** = `cargo build -p grid-cli --all-features` 干净退出 (exit 0, 0 warnings 不强求 — 与 default + studio build 一致),AND `cargo build --workspace` 不引入新警告。**不跑** `cargo test --all-features` (per CLAUDE.md no-full-tests rule)。

### CLI-X2 — `delete_session` + `export_session` typed exit code

- **D-04 (X2 pattern)**: **复制 kill_session 模式逐字粘贴**, 不抽 helper。理由:
  - 三个 callsite (kill / delete / export) 同形但只是三个,karpathy `no premature abstraction` 与 v3.2 intentional light triage 精神一致
  - 每 site ~5 LOC, 比 helper 抽取后的 1 行调用 + helper 定义更清晰可读
  - Phase 5.5 Plan 01 Task B1 已建立的 `main.rs:92-96` `downcast_ref::<GridError>` arm 自动 catch 新的两处, 无需 main.rs 改动

- **D-05 (X2 delete_session 具体改法)**: 当前 `delete_session` L95-103 有 **两个 bug** (不是一个):
  1. `eprintln!` 写 stderr 但仍 `Ok(())` 返回 → **exit 0** (silent success despite no-op) — 严重于 `export_session` 的 anyhow 退 1
  2. 没有 `get_session` 前置 check,直接 `delete_session()` 返回 bool false 时 "not found" 不会触发 typed error
  - 修法: 复制 kill_session L113-120 的前置 `get_session(&sid).await.is_none() → return Err(GridError::session_not_found(session_id).into());` block 到 delete_session 顶部;保留下方 `if session_store.delete_session(&sid).await { println! }` block,但删除 else 分支的 eprintln (那条路径不再可达)

- **D-06 (X2 export_session 具体改法)**: L157 `.ok_or_else(|| anyhow::anyhow!("Session not found: {}", session_id))?` → `.ok_or_else(|| anyhow::Error::from(crate::error::GridError::session_not_found(session_id.clone())))?` (或显式 `match` 转 `GridError::session_not_found(session_id).into()` 后 `?`);保留 L165-176 的 format 分发逻辑不变。

### Tests

- **D-07 (X2 test scope)**: 新增 2 个 integration test 到 `crates/grid-cli/tests/cli_integration.rs`:
  - `test_delete_nonexistent_session_exits_4` — 完全复制 `test_kill_nonexistent_session_exits_4` 结构 (`grid_bin().args(["session", "delete", "does-not-exist-xyz"]).output()` + `assert_eq!(out.status.code(), Some(4), ...)`)
  - `test_export_nonexistent_session_exits_4` — 同上 (`["session", "export", "does-not-exist-xyz"]`);format 参数不传, 让 export 走默认 json (但因 not-found 在 format match 前抛出, 无关紧要)
  - **不加 stderr message assertion** (匹配 Phase 5.5 NEW-A3 既有测试精神,避免过紧约束 GridError 的 Display 实现细节)
  - 不引入 `test-case` crate (parametrize),保持与现有测试风格一致
