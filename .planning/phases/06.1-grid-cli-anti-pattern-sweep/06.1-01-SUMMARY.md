## Deviations from Plan

### 1. [Rule 2 - Auto-add missing scope] handler.rs L133 wit-bindgen accessor rename + host_impl.rs L3 doc comment

- **Found during:** Task 4 (Bucket A WIT rename)
- **Issue:** The plan's Bucket A enumeration covered the obvious `OctoHookPlugin` callsites at handler.rs:121 + L127 and the `octo::hook::host::Host` references at host_impl.rs:49 + L166. But the wit-bindgen package rename `octo:hook` → `grid:hook` also affects the package-name-derived accessor `plugin.octo_hook_hook_handler()` at handler.rs:133 (must become `plugin.grid_hook_hook_handler()`) and the doc-comment reference to `octo-hook.wit` at host_impl.rs:3. Without these renames, the build would have continued to fail.
- **Fix:** Renamed `plugin.octo_hook_hook_handler()` → `plugin.grid_hook_hook_handler()` at handler.rs:133; renamed `octo-hook.wit` → `grid-hook.wit` in the host_impl.rs:3 doc comment.
- **Files modified:** crates/grid-engine/src/hooks/wasm/handler.rs + crates/grid-engine/src/hooks/wasm/host_impl.rs
- **Commit:** Included in atomic Bucket A commit 97f59e5 (kept atomic per CONTEXT.md "wit/ rename must be a single commit" rule)

### 2. [Observation - Non-blocking] Default-feature `unused variable: config` warning

- **Found during:** Task 3 (Bucket B `_config` → `config` rename in sandbox/wasm.rs:334)
- **Issue:** Renaming `_config` → `config` introduces a `unused variable: config` warning in default-feature builds because the function body that uses `config.memory_limit`, etc., is gated behind `#[cfg(feature = "sandbox-wasm")]`. In default-feature mode the body is the short early-return, so `config` is unused.
- **Resolution:** Accepted as-is. The alternatives (`#[cfg_attr(not(feature = "sandbox-wasm"), allow(unused_variables))]`, or renaming back to `_config` and dropping the underscore inside the cfg block) all add ceremony for a cosmetic warning. The plan's verify command `cargo build -p grid-cli --all-features` passes; the workspace check passes; this warning is informational.
- **Action:** Documented here. No code change.

### 3. [Observation - Plan accuracy] Plan-text expected `eprintln!("Session not found:` count = 2, actual = 3

- **Found during:** Task 1 verification
- **Issue:** The plan's Task 1 acceptance criterion stated: "`grep -cF 'eprintln!("Session not found:' crates/grid-cli/src/commands/session.rs` returns 2 (one in delete_session before the typed-error return, one preserved in kill_session — the dead else-arm at old L102 is REMOVED)". Actual count is 3 because there is a pre-existing `eprintln!("Session not found: ...")` in show_session at L88, which the plan author overlooked.
- **Resolution:** show_session is out of scope for CLI-X2 (it doesn't have an exit-code anti-pattern; it prints not-found message but the show command is informational). The semantic invariant the plan was checking — "dead else-arm in delete_session is removed" — is satisfied. The three sites now are: show_session (pre-existing informational), delete_session (new, before typed return), kill_session (pre-existing, before typed return). All correct.
- **Action:** Acceptance criterion interpretation: the underlying invariant holds. No code change. Filed as plan-text inaccuracy for future planner reference.
