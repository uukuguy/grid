## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Case sensitivity broke fuzzy matching**
- **Found during:** Task 3 (testing)
- **Issue:** rapidfuzz is case-sensitive by default. "Deploy SCADA" vs "deploy scada" produces low token_sort_ratio scores (0.39 vs expected 0.76+).
- **Fix:** Normalize both index entries and query to lowercase before matching.
- **Files modified:** `nlu_resolver.py`
- **Commit:** `1cf37a2e`

**2. [Rule 1 - Bug] Description-augmented index entries diluted token_sort_ratio**
- **Found during:** Task 3 (testing)
- **Issue:** Concatenating `name: description` for the search index caused token_sort_ratio to score "deploy the scada calibration" → "threshold calibration: scada threshold calibration verification skill" at 0.49 (below 0.6 threshold). WRatio was too permissive (0.86 for unrelated queries).
- **Fix:** Use name-only matching (lowercased) in the index. token_sort_ratio with name-only gives 0.61 for realistic matching and 1.00 for exact matches, while keeping unrelated queries at 0.33.
- **Files modified:** `nlu_resolver.py`
- **Commit:** `1cf37a2e`

**3. [Rule 1 - Bug] Pydantic validation blocked empty skill_id**
- **Found during:** Task 3 (integration testing)
- **Issue:** `IntentDispatchRequest.skill_id` had `min_length=1`, so empty strings were rejected before reaching the NLU branch (HTTP 422 instead of NLU resolution).
- **Fix:** Changed `skill_id` `min_length` from `1` to `0` to allow empty strings through to the NLU logic.
- **Files modified:** `api.py`
- **Commit:** `1cf37a2e`

**4. [Rule 2 - Missing Critical Functionality] Duplicate skill name handling**
- **Found during:** Task 1 (implementation review)
- **Issue:** If two skills have the same name, `build_index_from_list()` would silently overwrite the first entry's skill_id in the index.
- **Fix:** When a duplicate display_text is detected, append the skill_id as a disambiguation suffix (e.g., `threshold calibration (skill.verification.xyz)`).
- **Files modified:** `nlu_resolver.py`
- **Commit:** `1cf37a2e`
