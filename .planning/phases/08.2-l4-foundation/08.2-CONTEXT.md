<decisions>
## Implementation Decisions

### D34 — Intent→Skill NLU (rapidfuzz matching)
- **D-01:** Use `rapidfuzz>=3.9` (`pip install rapidfuzz`) for fuzzy string matching.
- **D-02:** Match intent_text against skill name + description index. Use `rapidfuzz.process.extractOne()` with token sort ratio for typo-tolerant matching.
- **D-03:** Skill registry client already exists at `l1_client.py`. NLU module queries skill list → builds index → matches intent → dispatches to best skill.
- **D-04:** Fallback: if no skill above confidence threshold (0.6), return ranked list for user disambiguation.
- **D-05:** ~100-150 LOC in new `nlu_resolver.py` module + integration in `api.py` `dispatch_intent` endpoint.

### D38 — user_id tenant isolation
- **D-06:** `L2Client.search_memory()` passes `user_id` from session context. L2 API already accepts user_id — verify at search endpoint.
- **D-07:** ~15 LOC — find the L2Client call site, add `user_id` param.

### D41 — Session list endpoint
- **D-08:** New REST endpoint `GET /v1/sessions` returns paginated session list with metadata.
- **D-09:** CLI `eaasp-cli-v2 session list` wired to call this endpoint.
- **D-10:** ~30 LOC — endpoint handler + CLI command wiring.

### Safety items — copy L3 patterns
- **D-11:** D28 exception handler: Global `@app.exception_handler(Exception)` catching all → JSON `{"error": ..., "detail": sanitize_errors(exc.errors())}`. Copy D22 from L3 `api.py`.
- **D-12:** D29 path param validation: `session_id` validator with regex + 404 for missing. Copy D18 from L3.
- **D-13:** D31 loguru init: `logger.configure(handlers=[...])` in lifespan, level from env. Copy D23 from L3.
- **D-14:** D39 policy hash: `hashlib.sha256(json.dumps(policy, sort_keys=True).encode()).hexdigest()` instead of `str(int())`. ~3 LOC.

### the agent's Discretion
- NLU confidence threshold and disambiguation UX
- Session list pagination params (limit/offset defaults)
- Plan structure — D34 NLU as core task, safety items bundled

</decisions>
