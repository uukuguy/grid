- Add a private method `_check_burst_rate(self) -> None`:
  ```python
  def _check_burst_rate(self) -> None:
      """L4-14 / D125 — detect event burst >500 events/sec and warn."""
      now = time.time()
      window_start = now - 1.0
      # Clean old buckets
      stale = [t for t in self._burst_counter if t < window_start]
      for t in stale:
          del self._burst_counter[t]
      # Count events in the last second
      rate = sum(v for t, v in self._burst_counter.items() if t >= window_start)
      if rate > 500:
          logger.warning(
              "L4 event burst detected: %d events/sec (threshold=500). "
              "Events may be dropped if queue fills.",
              rate,
          )
  ```
- In `ingest()`, after `self._queue.put_nowait(event)`, increment the counter:
  ```python
  now = time.time()
  bucket = int(now)
  self._burst_counter[bucket] = self._burst_counter.get(bucket, 0) + 1
  ```
- Also call `_check_burst_rate()` after incrementing.
- Initialize `self._burst_counter: dict[float, int] = {}` in `__init__`.
- Add `import time` at the top of the file.

**L4-15 (D110): dependencies field `kind: runtime|intent` semantics distinction**

Currently `dependencies` is a flat list of strings like `["mcp:memory-store"]`. After this change, each dependency becomes `{"name": "...", "kind": "runtime"|"intent"}`.

In `context_assembly.py`, `_normalize_skill_instructions()` function (line 89-101):
- Change the `"dependencies"` normalization to handle both old (string) and new (dict) formats:
  ```python
  # L4-15 / D110 — normalize dependencies to {name, kind} dicts.
  # Backward-compat: flat string deps default to kind="runtime".
  raw_deps = raw.get("dependencies") or []
  normalized_deps: list[dict[str, str]] = []
  for dep in raw_deps:
      if isinstance(dep, str):
          # Legacy flat string → default kind="runtime"
          normalized_deps.append({"name": dep, "kind": "runtime"})
      elif isinstance(dep, dict):
          name = dep.get("name") or dep.get("dep") or ""
          kind = dep.get("kind", "runtime")
          if kind not in ("runtime", "intent"):
              kind = "runtime"  # Invalid kind → default
          if name:
              normalized_deps.append({"name": name, "kind": kind})
      # Silently skip malformed entries (no name).
  ```
- Return `"dependencies": normalized_deps` instead of `list(raw.get("dependencies") or [])`.

In `session_orchestrator.py`, `create_session()` Step 6 (lines 314-316):
- Update the MCP resolution to work with the new format:
  ```python
  # L4-15 / D110 — deps are now {name, kind} dicts.
  # MCP deps start with "mcp:" in name field, kind="runtime".
  raw_skill_deps = skill_instructions.get("dependencies") or []
  ```

**Phase 3+ documented schema change:** This is a backward-incompatible schema change. Any existing skill with flat `dependencies: ["mcp:memory-store"]` in its frontmatter will be normalized to `[{"name": "mcp:memory-store", "kind": "runtime"}]` at the L4 boundary. Skill manifests stored in the L2 registry (with their raw YAML) are NOT changed — only the L4 in-memory representation is normalized.

**L4-16 (D33): SESSION_CREATED event payload reference-mode dedup**

Currently in `session_orchestrator.py`, `create_session()` (lines 259-271), the SESSION_CREATED event stores the full SessionPayload:
```python
await db.execute("""
    INSERT INTO session_events
        (session_id, event_type, payload_json, created_at)
    VALUES (?, ?, ?, ?)
    """,
    (session_id, "SESSION_CREATED",
     json.dumps({"payload": payload}, sort_keys=True),  # ← DUPLICATE
     created_at))
```

The full payload is already stored in the `sessions` table (`payload_json` column, line 255). This is a complete duplicate (~2-5 KB per session).

**Change:** Store a lightweight reference instead:
```python
# L4-16 / D33 — SESSION_CREATED event stores reference, not full payload.
# Full payload lives in sessions.payload_json (line above); event links back.
await db.execute("""
    INSERT INTO session_events
        (session_id, event_type, payload_json, created_at)
    VALUES (?, ?, ?, ?)
    """,
    (session_id, "SESSION_CREATED",
     json.dumps({
         "session_id": session_id,
         "reference_mode": True,
         "payload_keys": list(payload.keys()),
     }, sort_keys=True),
     created_at))
```

**Migration note:** Existing SESSION_CREATED events in the DB keep their full payload. New events going forward use reference mode. Consumers that read SESSION_CREATED events and expect `payload.payload.field` should:
- If `payload.reference_mode` is True → read from the sessions table instead
- If `payload.payload` exists → legacy format (no migration needed, backward-compat)
