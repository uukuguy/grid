## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| L1 runtime → L4 orchestration (gRPC) | grid-runtime emits SendResponse with new ChunkType wire values; L4 must not inject unsafe content from THINKING_TRACE / ATTACHMENT_REF payloads into UI/SSE without escaping |
| L1 runtime → L3 governance (hook envelope) | SubagentStart payload carries `subagent_name`, `purpose` — user-controlled strings flowing into governance scripts |
| L1 runtime → L2 trajectory (Hook event log) | TaskCheckpoint metadata logged for audit — string fields could carry injection vectors into log analysis tools |
| Test harness mock_openai_server.py → grid-runtime probe | Mock-driven SSE delta — production must not relax parser strictness based on mock fixture shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5.3-01 | Tampering | SubagentStartHook payload (`subagent_name`, `purpose` free-form strings) | mitigate | proto3 strong-typed strings; serde_json escapes control chars at envelope serialization; ADR-V2-006 §2.3 top-level shape preserved (Pitfall 3 guard). Length cap deferred (no UX evidence of need yet). |
| T-5.3-02 | Information Disclosure | THINKING_TRACE payload (reasoning_summary may leak internal CoT to user-facing event stream) | mitigate | L4 `_chunk_type_to_wire` is descriptor-driven; downstream L4 SHOULD gate THINKING_TRACE on user-facing visibility flag (out of scope for 5.3 wire-only contract). Capability not requirement per D-02 — runtime can choose not to emit. |
| T-5.3-03 | Tampering | ATTACHMENT_REF URI carries opaque user-influenceable string (potential path traversal `file://../../etc/passwd`) | accept | Per D-03, backend (URI scheme whitelist + blob storage adapters) is OUT OF SCOPE for 5.3 — wire only carries opaque string; consumer-side validation deferred to next milestone.
