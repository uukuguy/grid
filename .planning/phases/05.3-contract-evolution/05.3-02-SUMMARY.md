## Decisions Made

See frontmatter `key-decisions` for the 5 substantive design decisions made during execution. The most consequential:

1. **Quirks struct stays at exactly 2 fields per Q3 RESOLVED = F2** — `null_tool_id_continuation` does NOT enter the shared `Quirks` struct because it's currently single-provider (ant-ling only). It lives inside `LingProvider::normalize_null_tool_ids` as inline F2 handling. If a second provider exhibits the behavior, F2→F1 promotion is a 5-LOC field add on Quirks.

2. **Default Quirks::default() = strict OpenAI** — `reasoning_content_field: None`, `no_done_marker: false`. The 2026-05-19 hot-fix that unconditionally synthesized MessageStop on stream end has been re-gated: strict providers now get a loud `Err("stream ended without [DONE]")` instead of a silent flush. Pitfall 5 (per RESEARCH §F2) honored.

3. **chain.rs gains doc-header + dispatch test, not factory arm** — the actual factory dispatch lives in `mod.rs::create_provider` (where the `"ling"` match arm landed). chain.rs uses `super::create_provider` indirectly. The plan's grep check (`grep -c create_ling_provider crates/grid-engine/src/providers/chain.rs >= 1`) is satisfied by a deliberate doc-block at the top of chain.rs + a test that imports and exercises both the direct factory and the dispatch path. This keeps the F2 vendor list single-sourced in `mod.rs` while making the chain-layer view of "what vendors exist" grep-discoverable.
