**D101 (L2-16) — FastAPI HTTPException response shape:**
In `api.py`:
1. Add imports at top: `from fastapi import Request` and `from starlette.responses import JSONResponse`
2. Add a custom HTTPException handler inside `create_app()` (before `return app`):

```python
@app.exception_handler(HTTPException)
async def http_exception_handler(request: Request, exc: HTTPException) -> JSONResponse:
    """Flatten nested detail-dict responses to match the L2 flat contract.

    FastAPI's default handler wraps detail in {"detail": value}, producing
    {"detail": {"code": ..., "message": ...}} when detail is a dict.
    The L2 contract expects {"code": ..., "message": ...} directly.
    """
    if isinstance(exc.detail, dict):
        return JSONResponse(status_code=exc.status_code, content=exc.detail)
    return JSONResponse(status_code=exc.status_code, content={"detail": exc.detail})
```

**D92 (L2-15) — MockEmbedding seed collision fix:**

In `provider.py` L160, the MockEmbedding.embed() method currently:
```python
seed = int.from_bytes(digest[:8], "little", signed=False)
```

This uses only the first 8 bytes of the SHA-256 digest as seed, limiting randomness to 2^64 states. For 1024-dimensional vectors, this creates collision risk in test suites with many embeddings.

Change to use all 32 bytes:
```python
seed = int.from_bytes(digest, "little", signed=False)
```

The `int.from_bytes()` call handles the full 32-byte (256-bit) digest natively. No other changes needed — the rest of the method (gauss sampling, normalization) works identically.

Note: This changes existing MockEmbedding outputs. All existing tests that compare against hardcoded mock embedding values will break. After making this change, update any test that hardcodes mock embedding values, or use the test's existing comparison mechanism if it uses computed reference values.
