# Phase 03.8.1 — Login + Refresh + Logout + Audit stamping

## Goal (1 sentence)

Add `POST /api/v1/auth/{login,refresh,logout}` HTTP endpoints backed by an in-process
credential store (`UserStore`) and token blacklist (`TokenBlacklist`); extend the existing
audit middleware to stamp tenant_id and role on every authenticated request by reading
`Extension<JwtClaims>` set by 03.8.0's Full-mode middleware.

## Why this scope

03.8.0 minted the JWT, but it's invisible — there's no path for a user to obtain one.
03.8.1 surfaces it. The endpoints are intentionally minimal:
- **login** = mint fresh JWT from credential
- **refresh** = mint new JWT before old one expires (no DB session needed)
- **logout** = blacklist the presented token's `jti` (server-side TTL map)

Cross-tenant scoping (RBAC + TenantContext::for_multi_user) is **NOT** in 03.8.1; it lands in 03.8.2.
AUDIT-01 *is* in 03.8.1 because audit stamping is a single-threaded infrastructure change that
mostly precedes RBAC — the cross-tenant gating in 03.8.2 reads the audit log to confirm tenant context.

## Locked decisions (consistent with 03.8.0)

| # | Decision | Source |
|---|----------|--------|
| D-01 | UserStore is in-process (HashMap-backed, seeded from `GRID_USERS_JSON` env or YAML); DB persistence defers to v3.9+ | scope discipline; v3.8 ships the endpoints, not the user-management UI |
| D-02 | Password hashing = Argon2id; reuse existing `grid-engine::secret::vault` | no new crypto, leverages in-deps library |
| D-03 | Token blacklist is in-memory `Arc<Mutex<HashMap<String, Instant>>>`; entries auto-expire after TTL (lazy GC); DB-backed blacklist is v3.9+ | minimum viable; multi-instance deployments cluster-share via DB later |
| D-04 | Refresh path = mint new JWT with same claims + new exp; **no refresh-token concept in v3.8** (single token with sliding expiration) | minimum viable; refresh-rotation is v3.9+ |
| D-05 | Login response is `{ "access_token", "token_type": "Bearer", "expires_at" }`; body never leaks which credential failed (AUTH-04) | OWASP cheat-sheet-compliant |
| D-06 | Logout adds the *current request's* bearer token to blacklist; doesn't require body (token in `Authorization` header is sufficient) | minimal body |
| D-07 | Existing audit middleware signature unchanged; we read `Extension<JwtClaims>` to enrich the audit event | additive |
| D-08 | Add `tenant_id: Option<String>` + `role: Option<String>` to `AuditEvent` struct + DB columns; backfill OldEvents with NULL (migration `ADD COLUMN ... DEFAULT NULL`) | additive DB schema |
| D-09 | All error paths return 401 with `{ "error": "auth_failed", "message": "...safe..." }` — never disclose user existence | AUTH-04 contract |
| D-10 | No raw-password logging anywhere — even at debug level | feedback_no_hack_for_validation |

## Out of scope (deferred)

- DB-backed `users` table (D-01)
- DB-backed token blacklist (D-03)
- Refresh-token rotation (D-04)
- User-management endpoints (`POST /users`, `PATCH /users/:id`, `DELETE /users/:id`) — v3.9+
- Password reset / forgot-password — v3.9+
- 2FA / WebAuthn — never (out of project scope)
- Rate limiting on `/auth/login` — v3.9+

## Tasks

### Task 1: UserStore (in-process credential store)

**New file:** `crates/grid-server/src/auth/user_store.rs`

```rust
pub struct UserRecord {
    pub user_id: String,
    pub tenant_id: String,    // v3.8: every user belongs to exactly one tenant
    pub email: String,
    pub role: Role,           // reuses grid-engine::auth::Role
    pub password_hash: String, // Argon2id PHC string
}

pub struct UserStore {
    by_email: HashMap<String, UserRecord>,  // email -> record (login lookup)
    by_user_id: HashMap<String, UserRecord>,
}

impl UserStore {
    pub fn from_env() -> Result<Arc<Self>, String> {
        // Read GRID_USERS_JSON env var:
        //   [{"email":"a@x","password":"...","tenant_id":"x","user_id":"u1","role":"user"}, ...]
        // Argon2id-hash each password at load.
        // FAIL if missing AND mode=full (strict).
        // Single-user mode: succeed with empty store (login disabled).
    }

    pub fn verify_credentials(&self, email: &str, password: &str) -> Option<&UserRecord> {
        let r = self.by_email.get(email)?;
        if argon2::verify_password(password.as_bytes(), &r.password_hash).is_ok() {
            Some(r)
        } else {
            None
        }
    }
}
```

**Modifies:** `crates/grid-server/src/auth/mod.rs` (new module), `crates/grid-server/src/lib.rs` (re-export if needed)
**deps:** `argon2` already in workspace via `grid-engine::secret::vault`

### Task 2: TokenBlacklist (in-memory, TTL-based lazy GC)

**New file:** `crates/grid-server/src/auth/token_blacklist.rs`

```rust
pub struct TokenBlacklist {
    entries: Mutex<HashMap<String, Instant>>,  // jti string → expires_at
    default_ttl: Duration,                      // for evict()
}

impl TokenBlacklist {
    pub fn new(default_ttl: Duration) -> Self;

    /// Add a jti to the blacklist until `expires_at` (token's exp claim).
    pub fn blacklist(&self, jti: &str, expires_at: i64);

    /// Check if a jti is on the blacklist. Lazy-GCs stale entries while holding the lock.
    pub fn is_blacklisted(&self, jti: &str) -> bool;
}
```

**Note:** v3.8 JWT does not currently emit a `jti` claim. We need to ADD one in Task 5 below
(it's tiny — `pub jti: String` of `Uuid::new_v4().to_string()`).

### Task 4: HTTP endpoint handlers

**New file:** `crates/grid-server/src/api/auth.rs`

```rust
#[derive(Deserialize)] struct LoginRequest { email: String, password: String }
#[derive(Serialize)] struct LoginResponse { access_token: String, token_type: &'static str, expires_at: i64 }
#[derive(Serialize)] struct ErrorResponse { error: &'static str, message: String }

/// POST /api/v1/auth/login
/// Validates credentials; on success mints a JWT and returns LoginResponse.
/// On failure returns 401 with safe ErrorResponse (AUTH-04 / D-09).
pub async fn login_handler(
    State(state): State<AppState>,
    Json(req): Json<LoginRequest>,
) -> Result<Json<LoginResponse>, (StatusCode, Json<ErrorResponse>)>;

/// POST /api/v1/auth/refresh
/// Reads `Authorization: Bearer <old>`, validates it (jti not blacklisted, not expired),
/// then mints a NEW JWT with same claims + new exp + new jti. Old token remains valid
/// until its exp (no rotation in v3.8 per D-04).
pub async fn refresh_handler(
    State(state): State<AppState>,
    req: Request<Body>,
) -> Result<Json<LoginResponse>, (StatusCode, Json<ErrorResponse>)>;

/// POST /api/v1/auth/logout
/// Reads `Authorization: Bearer <token>`, validates it, extracts jti+exp,
/// blacklists jti until exp. Returns 204 No Content.
pub async fn logout_handler(
    State(state): State<AppState>,
    req: Request<Body>,
) -> Result<StatusCode, (StatusCode, Json<ErrorResponse>)>;
```

**Modifies:** `crates/grid-server/src/api/mod.rs` (re-export), `crates/grid-server/src/router.rs`
(wire routes), `crates/grid-server/src/state.rs` (add `users: Arc<UserStore>`,
`blacklist: Arc<TokenBlacklist>` to `AppState`)

## Risks

| Risk | Mitigation |
|------|-----------|
| Argon2 hashing on every login is slow (~50-300ms) | Test users use pre-computed hashes via `UserStore::from_json_raw` test helper; production path verified once |
| TokenBlacklist grows unbounded if logout floods | Lazy GC during `is_blacklisted` (auto-expire on read); cap is bounded by `# unique sessions × max-lifetime` |
| Tests that read the audit DB after a request need to wait for `tokio::spawn` to land | 50-100ms sleep in `audit_rows_carry_tenant_id_and_role`; or use `tokio::time::pause()` if available |
| Existing AuditEvent serialization breaks 03.8.0 callers | All fields are `pub` and additive; existing callers don't set the new fields → JSON `null`/`None` in their output |
