<decisions>
## Implementation Decisions

### D48 — ScopedHookBody proto change (full)
- **D-01:** Add `matcher` (string, regex/glob pattern) and `tool_filter` (repeated string, tool name allowlist) fields to `ScopedHookBody` in `proto/eaasp/runtime/v2/hook.proto`.
- **D-02:** Full proto → codegen cascade: proto change → `protoc` → Rust/Python bindings regeneration → grid-runtime hook dispatch reads new fields.
- **D-03:** Backward-compatible — new fields are optional (proto3 default behavior). Existing hooks without these fields work unchanged.
- **D-04:** ~50 LOC proto + ~20 LOC dispatch wiring.

### D50 — Prompt executor loop (reuse session provider)
- **D-05:** Use the session's existing LLM provider config (ANTHROPIC_API_KEY / OPENAI_API_KEY from env). Same provider as the agent session, separate call with yes/no system prompt.
- **D-06:** Prompt executor sends a compact decision prompt: "Given this hook context + tool use, should the tool be allowed? Reply YES or NO only."
- **D-07:** Response format: parse "YES"/"NO" from first token. Timeout: 10s default, configurable.
- **D-08:** ~80-120 LOC in hook executor module.

### Mechanical items (no decisions)
- **D65**: MCP connection pool — pre-warmed connections for multi-tool workloads. ~100 LOC.
- **D95**: FTS hit semantic score backfill from DB `embedding_vec` blob when HNSW add fails. ~40 LOC.
- **D100**: `write()`/`confirm()`/`archive()` surface `embedding_model` + dimension. ~15 LOC.
- **D108**: bats + shellcheck regression tests wired into `make verify`. ~60 LOC.
- **D105**: `HookPoint::ContextDegraded` deprecation warning + docs. ~5 LOC.
- **D107**: shared jq fragment extracted to `_lib/json_guards.sh`. ~15 LOC.

### the agent's Discretion
