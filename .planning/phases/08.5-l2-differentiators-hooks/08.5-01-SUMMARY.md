## Implementation Details

### Task 1: Connection Pool (D65)

- **`mcp_server.py`**: Added `_EmbeddingClientPool` class with lazy-init `httpx.AsyncClient` (keep-alive pooling: 5 max keepalive, 20 max connections), health-check (detects closed client, recreates), `close()` cleanup. Wired into `build_server()`: pool created, client injected via `set_embedding_client()` on first dispatcher init. Cleanup in `_serve_stdio()` finally block.
- **`provider.py`**: Added `_SHARED_CLIENT` module-level variable and `set_embedding_client()` setter. `OllamaEmbedding.embed()` checks for shared client and reuses it; falls back to one-shot client when not set (backward-compatible).
- **6 tests**: Pool unit tests (instance identity, dead-recreate, close, post-close reopen) + module setter test + integration test (two writes reuse same client).

### Task 2: Semantic Backfill (D95)

- **`index.py`**: After union merge (FTS + HNSW), identify FTS-only hits (`mem_id` in `fts_candidates` but not `hnsw_candidates`). Fetch `embedding_vec` blobs from DB (single `IN (...)` query, ordered by version DESC). Compute cosine similarity between `query_embedding` and unpacked vector. Clamp to [0, 1]. Update union entry with backfilled score. Graceful degrade: NULL blobs Ôćĺ score stays 0.0; keyword-only mode Ôćĺ skipped; unpack errors Ôćĺ caught and skipped.
- **`unpack_embedding`** import added to `index.py`.
- **4 tests**: Same-text cosine Ôëł 1.0, NULL vec Ôćĺ 0.0, HNSW hits preserved, keyword-only no-crash.

### Task 3: Embedding Surface (D100)

- **`files.py`**: Added `embedding_model_id=embedding_model_id` and `embedding_dim=embedding_dim` to `MemoryFileOut(...)` constructor in `write()`. Both `confirm()` and `archive()` inherit via shared `write()` return path. Variables already in scope from embedding compute step (lines 96ÔÇô107).

## Decisions Made

1. **Pool injection via module-level setter**: Instead of threading `httpx.AsyncClient` through `McpToolDispatcher Ôćĺ HybridIndex/MemoryFileStore Ôćĺ get_embedding_provider()`, used `set_embedding_client()` module-level setter in `provider.py`. Simpler, fewer files touched, same outcome.
2. **Backfill placement: after union merge, before ranking**: Most natural insertion point. Union already has FTS-only vs HNSW classification. Backfill modifies union entries before SearchHit creation.
3. **HNSW test expectation update**: `test_graceful_degrade_on_model_id_mismatch` now expects `semantic_score > 0.0` (not `== 0.0`) because D95 backfills from DB even when HNSW is empty. This is an improvement, not a regression.
