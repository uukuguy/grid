## Task Details

### Task 1: Prompt Executor YES/NO Mode (D50 / HOOK-03)

**TDD cycle:** RED → GREEN (no refactor needed).

**Implemented:**
- `execute_yesno_prompt()` — renders AI-SPEC "security guard" template, calls LLM with 100 max_tokens / temperature 0.0, retries once on timeout/error (2 attempts total), safe default DENY on all failures
- `parse_yesno_from_text()` — extracts first non-empty word, case-insensitive YES/NO match, safe default DENY for unrecognized input
- `HookActionConfig::YesNoPrompt { timeout }` — config variant with timeout from `HOOK_PROMPT_EXECUTOR_TIMEOUT_SEC` env var (default 10s)
- Bridge wiring: `DeclarativeHookBridge` dispatches `YesNoPrompt` actions to `execute_yesno_prompt()`

**Test coverage (11 new tests):**
- 7 unit tests: YES→allow, NO→deny, lowercase, whitespace handling, invalid word→deny, empty→deny, newlines
- 2 config deserialization tests: explicit timeout, default timeout
- 4 integration tests with `MockProvider`: YES allows, NO denies, unparseable denies, timeout retries then denies

**Key design decisions:**
- Safe default is DENY (unlike existing JSON executor which defaults to allow)
- Stateless call — does not use session's main conversation context
- No new API key requirements — reuses session provider credentials

### Task 3: bats + shellcheck CI Gate (D108 / HOOK-01)

**Modified:** `scripts/test_hook_scripts.sh` (rewritten, 107→86 LOC), `Makefile`

**Script features:**
- `--lint-only` — shellcheck only (lints `*.sh` under `examples/skills/*/hooks/`)
- `--test-only` — bats only (runs `*.bats` under `examples/skills/*/hooks/`)
- Default (no flags) — shellcheck first, then bats
- Conditional skip: missing shellcheck/bats prints "SKIP" and returns 0

**Makefile integration:**
- `hook-scripts-test` target calls `bash scripts/test_hook_scripts.sh || true`
- Wired as prerequisite of `make verify` (runs first, before Rust/TS checks)
- `|| true` ensures `make verify` never breaks on missing shellcheck/bats

**Existing test coverage:** 4 bats files, 22 tests across threshold-calibration and skill-extraction hooks.
