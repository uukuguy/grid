# AI-SPEC — Phase 8.5: Prompt Executor (D50 / HOOK-03)

**Created:** 2026-06-10
**Phase:** 08.5-l2-differentiators-hooks
**REQ-ID:** HOOK-03

---

## 1. What It Does

The Prompt Executor is an LLM-driven meta-guard that runs during hook execution. When a `ScopedHookBody::Prompt` hook fires (before/after a tool use), the executor calls an LLM to decide: should this tool use be allowed?

**Flow:** Hook fires → Prompt executor formats decision prompt → LLM call → parse YES/NO → allow or deny the tool.

## 2. Model Selection

- **Provider:** Reuse the session's existing LLM provider (ANTHROPIC_API_KEY or OPENAI_API_KEY from env). Per CONTEXT.md D-05.
- **Model preference:** Use a fast model suitable for yes/no decisions (≤2s latency target):
  - Anthropic: `claude-3-haiku-20240307` (fastest)
  - OpenAI: `gpt-4o-mini` (fastest)
- **Fallback:** If the fast model is unavailable, fall back to the session's default model with a shorter max_tokens (100).
- **No dedicated API key** — reuses session provider credentials. No new env var.

## 3. Prompt Template

```
System: You are a security guard for an AI agent. Given a hook context and a tool that the agent wants to use, decide if the tool should be allowed. Reply with exactly one word: YES or NO.

Context:
- Session ID: {session_id}
- Skill: {skill_name}
- Hook: {hook_name}
- Tool name: {tool_name}
- Tool arguments: {tool_args}

Tool description: {tool_description}

Should this tool be allowed? Reply YES or NO.
```

## 4. Response Parsing

- Parse the first non-empty word from the LLM response
- Accept: "YES", "yes", "Yes" → allow
- Reject: "NO", "no", "No" → deny
- Invalid/missing: default to DENY (safe default)
- Log the decision + reasoning (if the model includes reasoning text)

## 5. Timeout & Retry

- **Timeout:** 10 seconds per call (configurable via `HOOK_PROMPT_EXECUTOR_TIMEOUT_SEC` env var, default 10)
- **Retry:** 1 retry on timeout or LLM error (total 2 attempts max)
- **On failure (both attempts):** DENY the tool (safe default)

## 6. Tests

- **Unit test:** Mock LLM response "YES" → allow, "NO" → deny, "MAYBE" → default deny
- **Unit test:** Timeout → retry → deny on second timeout
- **Unit test:** Empty response → default deny
- **Contract test:** Prompt executor smoke — hook fires, executor calls mock LLM, decision returned

## 7. Evaluation Criteria

| Criterion | Threshold |
|-----------|-----------|
| Decision accuracy (mock tests) | 100% (unit tests) |
| Timeout behavior | Deny on timeout, no hang |
| Safe default | Deny on any parse failure |
| Speed target | ≤2s for yes/no decision (best-effort, not enforced in tests) |
