    Create `scripts/_lib/json_guards.sh` with shared helper functions:

    ```bash
    #!/usr/bin/env bash
    # Shared jq validation guards for hook and verification scripts.
    # Source this file: source "$(dirname "${BASH_SOURCE[0]}")/../_lib/json_guards.sh"

    # Ensure jq is available; fail with a clear message if not.
    require_jq() {
        if ! command -v jq >/dev/null 2>&1; then
            echo "ERROR: jq is required but not installed. Install: brew install jq" >&2
            exit 1
        fi
    }

    # Check if a JSON file exists and is non-empty.
    json_file_valid() {
        local f="$1"
        [ -f "$f" ] && jq empty "$f" 2>/dev/null
    }

    # Count events in a JSON events file by event_type.
    json_event_count() {
        local f="$1" event_type="$2"
        jq --arg et "$event_type" '[.events[]? | select(.event_type==$et)] | length' "$f" 2>/dev/null || echo 0
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM response → YES/NO parser | Untrusted LLM output crosses into hook decision logic |
| Hook scripts (bats/shellcheck) → CI pipeline | Untrusted script content from skill definitions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.5-07 | Spoofing | prompt_executor.rs — LLM response parsing | mitigate | YES/NO parser ignores surrounding text; only first non-empty word considered. Safe default is DENY (not allow). Case-insensitive but exact match required. |
| T-08.5-08 | Denial of Service | prompt_executor.rs — slow LLM response | mitigate | 10s timeout + 1 retry (2 attempts max). Both failures → safe default DENY. Configurable via HOOK_PROMPT_EXECUTOR_TIMEOUT_SEC env var. |
| T-08.5-09 | Elevation of Privilege | prompt_executor.rs — prompt injection via tool_args | mitigate | Tool arguments are passed as literal strings in the prompt template — no template interpolation or eval. The LLM sees them as plain text context. |
| T-08.5-10 | Tampering | test_hook_scripts.sh — shellcheck | accept | shellcheck is a read-only lint tool; no execution risk. Conditional skip ensures no CI breakage in environments without it. |
| T-08.5-11 | Information Disclosure | prompt_executor.rs — session data in prompt | accept | Session ID and tool name/args are part of the hook context that would already be visible to hook handlers. No additional data exposure. |
</threat_model>
