From crates/grid-engine/src/hooks/declarative/config.rs (lines 23-34):
```rust
pub struct HookEntry {
    /// Regex or glob pattern to match tool names. `"*"` matches all.
    pub matcher: String,
    /// Optional condition using PermissionRule syntax.
    pub if_condition: Option<String>,
    /// Actions to execute when the matcher hits.
    pub actions: Vec<HookActionConfig>,
}
```

**Key insight**: `HookEntry.matcher` already exists at the ENTRY level (filters which tools a hook entry fires for). D48 adds `matcher` + `tool_filter` at the ScopedHookBody level — a per-body secondary filter that refines which specific tools a particular action body targets. This is additive and backward-compatible (absent = match everything at the body level, existing behavior).
