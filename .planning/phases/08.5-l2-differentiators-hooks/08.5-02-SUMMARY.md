# Phase 08.5 Plan 02: Hook Infrastructure — ScopedHookBody Targeting + ContextDegraded Deprecation

**One-liner:** Add `matcher` + `tool_filter` to `ScopedHookBody` proto/Rust types for per-body tool targeting; emit deprecation warning for `ContextDegraded` alias.

## Summary

Two hook infrastructure changes:

1. **D48 / HOOK-02 — ScopedHookBody fine-grained targeting**: Added `matcher` (regex/glob pattern) and `tool_filter` (tool name allowlist) fields to `ScopedHookBody` across proto schema and Rust skill parser. These are secondary filters applied AFTER the entry-level `HookEntry.matcher` has already matched. Both fields are optional (absent = match all, backward-compatible).

2. **D105 / HOOK-04 — ContextDegraded deprecation**: Split the `"PostCompact" | "ContextDegraded"` match arm in `hook_point_from_name()` so that using `"ContextDegraded"` emits a `tracing::warn!` deprecation warning while still resolving to `HookPoint::PostCompact`. Updated `PostCompact` doc comment with migration guidance.

## Tasks Executed

### Task 1: ScopedHookBody proto change — matcher + tool_filter (D48 / HOOK-02)
**Commit:** `e6b837a0`

- **Proto**: Added `ScopedHookBody` message to `hook.proto` with `string matcher = 1` and `repeated string tool_filter = 2`.
- **skill_parser.rs**: Wrapped the existing `ScopedHookBody` tagged enum into a new struct with `matcher`/`tool_filter` fields and a flattened `body: ScopedHookBodyKind`. This preserves backward compatibility — existing YAML without `matcher`/`tool_filter` deserializes with defaults (`None`/`vec![]`).
- **Tests**: Updated `substitute_scoped_hooks` and all test match arms to use `ScopedHookBodyKind::Command`/`Prompt` via `.body.body` access.
- **Verification**: Proto compiles via protoc. skill-registry compiles and passes 9/9 unit tests + 37/38 integration tests (1 pre-existing failure in `parse_skill_extraction_example_skill` unrelated to this change).

### Task 2: HookPoint ContextDegraded deprecation warning + docs (D105 / HOOK-04)
**Commit:** `b861a7a0`

- **runtime.rs**: Split `"PostCompact" | "ContextDegraded"` into separate arms. `"ContextDegraded"` now calls `tracing::warn!` before returning `Some(HookPoint::PostCompact)`.
- **hooks/mod.rs**: Enhanced `PostCompact` doc comment with explicit deprecation guidance for `"ContextDegraded"` alias.
- **Verification**: grid-engine compiles with no new errors. All 128 hooks tests pass.
