<decisions>
## Implementation Decisions

### D20 — eaasp_common shared package
- **D-01:** Create `tools/eaasp-common/` as a new minimal Python package. Contains `_sanitize_errors()` as the initial utility. L3 depends on it via `pip install -e`, following hatchling conventions.
- **D-02:** Structure: `tools/eaasp-common/pyproject.toml` + `src/eaasp_common/__init__.py` + `src/eaasp_common/errors.py`. Single module, no complex setup.
- **D-03:** L3 migration: remove local `_sanitize_errors()`, import from `eaasp_common.errors`. L2 can adopt in later phase (L2-12).

### D10 — rmcp MCP ServerHandler (Policy endpoints only)
- **D-04:** Dual-transport architecture — keep existing FastAPI REST endpoints, ADD MCP transport for policy endpoints only. Health (`/health`) and telemetry (`/telemetry/*`) stay REST-only.
- **D-05:** Policy MCP surface: `deploy_managed_hooks`, `switch_mode`, `list_versions`, `validate_session`. Follow L2's MCP server pattern (`tools/eaasp-l2-memory-engine/tests/test_mcp_server.py`).
- **D-06:** MCP transport mounted alongside REST in the same FastAPI lifespan. Uses `mcp` Python SDK (`pip install mcp>=1.27`). ~80-150 LOC.
- **D-07:** L2 tool manifest pattern (7 tools) is the reference — L3 MCP follows same naming convention.

### Small items (no decisions needed)
- **D16**: TOCTOU fix — `policy_engine.deploy()` reads `created_at` AFTER commit, not before.
- **D19**: `switch_mode()` validates hook_id exists before mode change (404 for unknown, not silent create).
- **D25**: In-process concurrent deploy unit test. Uses `asyncio.gather(Semaphore(2))`, no `time.sleep()` anti-pattern.

### the agent's Discretion
- Exact MCP tool naming and tool manifest structure
- Shared package versioning strategy (start at 0.1.0)
- Plan structure (D20 as pre-requisite task, D10 independent of D16/D19/D25)

</decisions>
