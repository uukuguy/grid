## 2. Architecture Patterns

### D10 — Dual-Transport MCP (REST preserved + MCP added)

**CRITICAL CONSTRAINT (PITFALLS.md Line 116):** D10 is a **dual-transport** change, NOT a replacement.
- Keep ALL existing FastAPI REST endpoints (`deploy_managed_hooks`, `switch_hook_mode`, `list_policy_versions`, `validate_session`, `health`, telemetry)
- ADD MCP transport for policy endpoints only: `deploy_managed_hooks`, `switch_mode`, `list_versions`, `validate_session`
- Health (`/health`) and telemetry (`/telemetry/*`) stay REST-only
- REST remains the default connection path for L4; MCP becomes available for future L1 runtimes

**Reference pattern: L2 MCP server** (`tools/eaasp-l2-memory-engine/src/eaasp_l2_memory_engine/mcp_server.py`):
```python
from mcp.server import NotificationOptions, Server
from mcp.server.models import InitializationOptions
from mcp.types import TextContent, Tool

SERVER_NAME = "eaasp-l2-memory"
SERVER_VERSION = "0.1.0"

_TOOL_MANIFEST: list[Tool] = [...]  # Converted from MCP_TOOL_MANIFEST entries

def build_server(db_path: str | None = None) -> tuple[Server, str]:
    server = Server(SERVER_NAME)
    # Lazy-init dispatcher on first tool call
    
    @server.list_tools()
    async def list_tools() -> list[Tool]:
        return list(_TOOL_MANIFEST)
    
    @server.call_tool()
    async def call_tool(name: str, arguments: dict[str, Any]) -> list[TextContent]:
        dispatcher = await _get_dispatcher()
        try:
            result = await dispatcher.invoke(name, arguments or {})
        except ToolError as exc:
            return [TextContent(type="text", text=json.dumps({...}))]
        return [TextContent(type="text", text=json.dumps(result))]
    
    return server, db
```

**L3 MCP adaptation:**
- 4 MCP tools: `deploy_managed_hooks`, `switch_hook_mode`, `list_policy_versions`, `validate_session`
- Mount in FastAPI lifespan (alongside REST) — NOT as separate process
- Policy engine + audit already available in `create_app()` scope
- Wire the same `policy: PolicyEngine` instance into MCP tool dispatch
- MCP transport: SSE or stdio; SSE mounts in FastAPI via Starlette route (follows L2 SSE pattern lines 127-171)

**Acceptance gate:**
- `make dev-eaasp` L3 starts cleanly on port 18083 (REST)
- `make v2-phase2-e2e` passes (L4→L3 REST calls still work)
- MCP tools testable via `call_tool` pattern (follows L2 `test_mcp_server.py`)

### D16 — TOCTOU Fix (INSERT ... RETURNING)

**Current code** (`policy_engine.py:76-93`):
```python
await db.execute("BEGIN IMMEDIATE")
cur = await db.execute(
    """INSERT INTO managed_settings_versions (...) VALUES (?, ?, ?)""",
    (payload_json, hook_count, mode_summary_json),
)
version = cur.lastrowid
# ❌ TOCTOU: separate SELECT after commit
cur2 = await db.execute(
    "SELECT created_at FROM managed_settings_versions WHERE version = ?",
    (version,),
)
row = await cur2.fetchone()
await db.commit()
```

**Fix:** Use `INSERT ... RETURNING` (SQLite 3.35+ supports RETURNING; Python 3.12+ aiosqlite supports it):
```python
await db.execute("BEGIN IMMEDIATE")
cur = await db.execute(
    """INSERT INTO managed_settings_versions (...)
    VALUES (?, ?, ?) RETURNING created_at""",
    (payload_json, hook_count, mode_summary_json),
)
row = await cur.fetchone()
await db.commit()
# version, created_at both from RETURNING — atomic read, no TOCTOU
```

~5 LOC change. Verify: `INSERT ... RETURNING` is supported in aiosqlite when SQLite ≥ 3.35 (confirmed: Python 3.12 bundles SQLite 3.40+).

async def switch_mode(self, hook_id: str, mode: str) -> ModeOverride:

### D20 — eaasp_common Shared Package

**Extract:** `_sanitize_errors()` from `api.py:268-286`:
```python
def _sanitize_errors(errors: list[dict[str, Any]]) -> list[dict[str, Any]]:
    """Strip non-JSON-serializable objects from Pydantic error dicts."""
    clean: list[dict[str, Any]] = []
    for err in errors:
        safe: dict[str, Any] = {}
        for key, value in err.items():
            if key == "ctx" and isinstance(value, dict):
                safe[key] = {
                    ctx_key: (
                        str(ctx_val) if isinstance(ctx_val, BaseException) else ctx_val
                    )
                    for ctx_key, ctx_val in value.items()
                }
            elif isinstance(value, BaseException):
                safe[key] = str(value)
            else:
                safe[key] = value
        clean.append(safe)
    return clean
```

### D25 — Concurrent Deploy Unit Test

**Design:** In-process, not multi-process E2E. Uses real SQLite (WAL mode supports concurrent writers).
```python
import asyncio

async def test_concurrent_deploy_no_data_loss(policy_engine: PolicyEngine) -> None:
    """Two coroutines deploy concurrently; both versions persist with correct counts."""
    async def deploy(n: int) -> int:
        s = _sample_settings(n, 0)
        r = await policy_engine.deploy(s)
        return r.version
    
    results: list[int] = []
    async with asyncio.TaskGroup() as tg:
        t1 = tg.create_task(deploy(5))
        t2 = tg.create_task(deploy(3))
    for t in (t1, t2):
        results.append(t.result())
    
    assert sorted(results) == [1, 2]
    versions = await policy_engine.list_versions()
    assert len(versions) == 2
    # Verify both payloads have correct hook counts
```

Constraints: NO `time.sleep()` anti-pattern. Uses `asyncio.TaskGroup` (Python 3.12+) or `asyncio.gather`.

## 3. Anti-Patterns & Warnings

### Don't Hand-Roll

- **MCP transport:** Don't implement MCP from scratch. Use the `mcp` Python SDK (`mcp.server.Server`, `mcp.types.Tool`, `mcp.server.stdio.stdio_server`).
- **SQL RETURNING:** Don't parse `lastrowid` + separate SELECT. Use `INSERT ... RETURNING`.
- **Packaging:** Don't use setuptools. Follow hatchling convention (matches all other EAASP tools).

### Common Pitfalls

1. **D10 replaces REST instead of adding alongside it → breaks all L4 consumers**
   - Gate: `make v2-phase2-e2e` MUST pass (validates REST endpoints still work)
   - Gate: `make dev-eaasp` L3 MUST start on port 18083

2. **D20 creates circular dependency (eaasp_common depends on L3)**
   - Verification: `eaasp-common` must NOT import anything from eaasp_l3_governance
   - `sanitize_errors()` uses only stdlib types (`dict`, `list`, `str`, `isinstance`)

3. **D25 uses `time.sleep()` to "wait" for concurrency**
   - This is an anti-pattern per PITFALLS.md. Use real asyncio concurrency.

4. **D16 changes the deploy() return shape**
   - The `DeployResult` model already has `created_at: str` — no API change, just the read mechanism
