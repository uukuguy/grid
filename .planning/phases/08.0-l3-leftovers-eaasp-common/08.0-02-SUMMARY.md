## What Changed

### D16 — Atomic deploy with INSERT ... RETURNING

**Before:** `deploy()` performed INSERT then a separate SELECT to read back `created_at`. A concurrent writer could insert between these two operations (TOCTOU window).

**After:** Single `INSERT ... RETURNING version, created_at` atomically writes the row and returns both values. Removed 9 lines (separate SELECT + `cur.lastrowid`), added 3 lines.

```python
# Key change — policy_engine.py:83-91
INSERT INTO managed_settings_versions (...) VALUES (?, ?, ?)
RETURNING version, created_at
```

### D19 — switch_mode validation

**Before:** `switch_mode()` accepted any `hook_id` string and silently created a mode override, even for hooks not in any deployed policy.

**After:** `switch_mode()` queries `latest_version()` first, extracts hook_ids from the payload, and raises `HookNotFoundError(hook_id)` for unknown hooks. The API layer catches this and returns HTTP 404.

Added `HookNotFoundError` exception class + validation block + 2 new test cases. Updated existing `test_switch_mode_upserts_override` to deploy a policy first (required by validation).

### D25 — Concurrent deploy unit test

New test `test_concurrent_deploy_no_data_loss` launches two concurrent `deploy()` calls via `asyncio.TaskGroup`. Verifies both versions persist with correct hook counts. No `time.sleep()` — uses real asyncio interleaving with WAL-mode SQLite.
