# Phase 3.7.1 — grid-cli 实战可用性补全 — Context

## Goal (from milestone)

`grid-cli` must let one person drive an agent end-to-end for realistic enterprise-style scenarios without manual CLI-flag tuning. Audit the current `grid` command surface (16 commands) against 3–5 realistic enterprise scenarios, identify the gaps between Activation-9.0 code quality and end-to-end runnability, close the gaps.

## Scope

- Touch `crates/grid-cli/` (and `grid-cli` workspace member config). Possibly touch `grid-server` if a CLI command exercises server endpoints (cross-check needed).
- Shared core (`grid-engine` / `grid-runtime` / `grid-types` / `grid-sandbox` / `grid-hook-bridge`) — only if a scenario-blocking bug surfaces; per ADR-V2-023 P1 must work for both legs.
- Out of scope: `web/`, `tools/eaasp-*/`, `grid-server` multi-user (handled in 3.7.2/3.7.3/v3.8).

## What needs to be discovered (do not pre-decide — discuss-phase will)

1. **Current `grid` command surface**: list 16 commands; map each to a real-world use case.
2. **3–5 实战 enterprise scenarios** to validate: candidates include
   - multi-step agent with tool use (e.g. agent reads a file, writes a file, calls a web API)
   - memory-driven session (agent recalls context from prior turn)
   - hook-driven governance (PreToolUse hook fires, agent paused for approval)
   - long-running streaming session with stop/resume
   - parallel agent runs / batch mode
3. **Gap audit**: for each scenario × command matrix, mark PASS / MANUAL TUNING / FAIL.
4. **Repair plan**: group failures into themes (config defaults, error reporting, command discoverability, doctor/repair coverage, eval bridge, MCP log streaming, config persistence).

## Decisions NOT yet locked (will be locked in discuss-phase)

- Exact 3–5 scenarios to validate.
- Whether to ship a "quickstart" doc + script, or improve in-CLI onboarding, or both.
- Whether the audit-then-fix loop is 1 plan (audit + fix in same plan) or 2 plans (audit separate, fix separate).

## Acceptance standard

- 3 documented enterprise scenarios PASS end-to-end without manual CLI-flag tuning.
- Documented in `docs/status/PRODUCTION_USABILITY_2026-XX-XX.md` walkthrough (new file pattern parallel to `PRODUCT_STATUS_2026-XX-XX.md`).
- No regression: `cargo test -p grid-cli` PASS; Activation-9.0 score preserved.
