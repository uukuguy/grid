# Plan 07.3-01: L3 RBAC + hardening

## Objective

Land L3 governance's 3 P2 items (D8 access_scope RBAC middleware, D9 skill_usage real telemetry with L2 fallback, D46 skill-registry namespace conflict guard) plus 5 P3 cleanups (D22 global exception handler, D23 loguru init, D17 hook_id KeyError guard, D18 session_id validation, D26 tiebreaker-based test ordering). Make L3 production-ready on auth, observability, and error-handling — the v3.3 milestone's final phase.

Strict-by-default enforcement per ADR-V2-028 lineage: all env-var overrides reject malformed values at startup (loguru level, L2_MEMORY_ENGINE_URL, L2_BUSY_TIMEOUT_MS). RBAC rejects missing/wrong scope with 403. Skill-registry rejects namespace-conflicting registrations at submit() time.

This plan closes the v3.3 milestone — after Phase 7.3, all 4 v3.3 phases (7.0+7.1+7.2+7.3) are complete.

## Task DAG

```
Wave 0 (infra):    T01 (loguru) ──→ T02 (exception handler, depends on logger)
Wave 1 (auth):     T03 (D46 skill-registry guard, independent) ∥ T04 (D8 RBAC middleware, after T02)
Wave 2 (telemetry): T05 (D9 skill_usage, after T04 — RBAC guards the new endpoint)
Wave 3 (cleanup):  T06 (D17 hook_id) ∥ T07 (D18 session_id) ∥ T08 (D26 tiebreaker)
```

<task id="T03" name="D46 — Skill-registry namespace conflict guard (Wave 1 auth)">
  <action>
  在 `tools/eaasp-skill-registry/src/store.rs` 的 `submit()` 中添加 namespace 冲突检测。

  Step 1 — 定位 `submit()` 方法 (在 L212 附近, 函数签名 `pub async fn submit(&self, req: SubmitDraftRequest) -> Result<SkillMeta>`)。

  Step 2 — 在 `submit()` 的 INSERT 之前添加 namespace check:
  ```rust
  // D46 / L3-03 — namespace conflict guard.
  // Reject if another skill already claims this access_scope.
  if let Some(ref scope) = access_scope {
      let scope_clone = scope.clone();
      let id_clone = m.id.clone();
      let conflict: Option<String> = self.db.call(move |conn| {
          let mut stmt = conn.prepare(
              "SELECT id FROM skills WHERE access_scope = ?1 AND id != ?2 LIMIT 1"
          )?;
          let mut rows = stmt.query_map(
              rusqlite::params![scope_clone, id_clone],
              |row| row.get::<_, String>(0),
          )?;
          Ok(rows.next().transpose()?)
      }).await.context("check namespace conflict")?;

      if let Some(existing_id) = conflict {
          return Err(anyhow::anyhow!(
              "namespace_conflict: access_scope '{}' is already claimed by skill '{}'. \
               Each access_scope can be owned by at most one skill",
              scope, existing_id
          ));
      }
  }
  ```

  Step 3 — 边界情况:
  - 同一 skill 重新注册 (相同 id + 相同 scope) → 允许 (UPDATE/DELETE 路径, 不是新 namespace 声明)
  - `access_scope` 为 `None` → 跳过检查 (无 scope 的 skill 不参与 namespace 隔离)
  - `access_scope` 为 `*` → 允许多个 skill (wildcard scope 的语义是 "无限制", 不适合独占)

  Step 4 — 验证:
  - 在 `tools/eaasp-skill-registry/tests/scope_filter.rs` (或新文件 `tests/namespace_conflict.rs`) 中添加测试:

    - `test_register_different_scope_allowed`: skill A scope=X, skill B scope=Y → 都成功
    - `test_register_same_id_same_scope_allowed`: skill A scope=X 重新注册 → 允许 (重新部署)
    - `test_register_null_scope_allowed`: skill A scope=None, skill B scope=None → 都成功 (无冲突)
  - `cargo test -p eaasp-skill-registry --test namespace_conflict` → all PASS
  </action>

  <acceptance>

<task id="T04" name="D8 — access_scope RBAC middleware (Wave 1 auth)">
  <action>
  实现 `require_access_scope()` FastAPI dependency, 在 `validate_session()` 中注入 RBAC 检查。

  Step 1 — 在 `api.py` 中添加 dependency function (放在 `create_app()` 外部, 作为模块级函数):
  ```python
  from fastapi import Header

  async def require_access_scope(
      x_session_scope: str | None = Header(default=None, alias="X-Session-Scope"),
  ) -> str:
      """Extract and return the caller's access_scope from X-Session-Scope header."""
      if x_session_scope is None:
          raise HTTPException(
              status_code=403,
              detail={
                  "error": "forbidden",
                  "message": "missing X-Session-Scope header — RBAC required",
              },
          )
      return x_session_scope
  ```

  Step 2 — 在 `validate_session()` 中集成:
  - 修改函数签名, 添加 `caller_scope: str = Depends(require_access_scope)`:
    ```python
    from fastapi import Depends

    @app.post("/v1/sessions/{session_id}/validate")
    async def validate_session(
        session_id: str,
        body: SessionValidateRequest,
        caller_scope: str = Depends(require_access_scope),
    ) -> dict[str, Any]:
    ```
  - 在 hook 匹配循环中, 如果 hook 声明了 `access_scope` 且 caller_scope 不是 wildcard `*`, 则检查:
    ```python
    hook_scope = hook.get("access_scope")
    if hook_scope is not None and caller_scope != "*" and hook_scope != caller_scope:
        logger.warning(
            "RBAC rejected",
            hook_id=hook.get("hook_id"),
            caller_scope=caller_scope,
            required_scope=hook_scope,
        )
        continue  # skip this hook — caller's scope doesn't match
    ```

  Step 3 — Wildcard 语义:
  - `caller_scope="*"` → 所有 hook 通过 (admin/debug 角色)
  - `hook_scope=None` → 跳过 RBAC 检查 (向后兼容 — 旧 hook 无 scope 声明)
  - `hook_scope != None and hook_scope != caller_scope and caller_scope != "*"` → 跳过该 hook

  Step 4 — 验证:
  - 在 `tests/test_session_validate.py` 中添加:
    - `test_validate_rejects_missing_scope_header`: 不带 X-Session-Scope header → 403
    - `test_validate_skips_hook_with_mismatched_scope`: scope=A 的 caller, scope=B 的 hook → hook 不返回
    - `test_validate_includes_hook_with_matching_scope`: scope=X 的 caller, scope=X 的 hook → hook 返回
    - `test_validate_wildcard_scope_includes_all`: scope=* 的 caller → 所有 hook 返回
    - `test_validate_hook_without_scope_included`: caller scope=X, hook 无 scope → hook 返回 (向后兼容)
  - `uv run pytest tests/test_session_validate.py -v` → 全部 PASS
  </action>

  <acceptance>
  - `require_access_scope()` dependency 从 `X-Session-Scope` header 提取 scope
  - 缺失 header → 403 `{"error": "forbidden", "message": "missing X-Session-Scope header"}`

  Step 2 — 在 `audit.py` 中添加 `skill_usage()` 方法:
  ```python
  async def skill_usage(
      self, skill_id: str, since: str | None = None, l2_base_url: str | None = None
  ) -> dict[str, Any]:
      """Return per-skill invocation stats. Primary: L2 audit log. Fallback: L3 local store."""
      # L2-primary path
      if l2_base_url:
          try:
              async with httpx.AsyncClient(timeout=5.0) as client:
                  resp = await client.get(
                      f"{l2_base_url}/v1/memory/{skill_id}/usage",
                      params={"since": since} if since else {},
                  )
                  if resp.status_code == 200:
                      data = resp.json()
                      data["source"] = "l2"
                      return data
          except (httpx.ConnectError, httpx.TimeoutException):
              pass  # fall through to L3 fallback

      # L3-fallback path
      db = await connect(self.db_path)
      try:
          cur = await db.execute(
              """SELECT COUNT(*) as invocations,
                        MIN(received_at) as first_seen,
                        MAX(received_at) as last_seen
                 FROM telemetry_events
                 WHERE hook_id = ? AND json_extract(payload_json, '$.skill_id') = ?""",
              (skill_id,) * 2,
          )
          row = await cur.fetchone()
      finally:
          await db.close()

      assert row is not None
      return {
          "skill_id": skill_id,
          "invocations": row["invocations"],
          "first_seen": row["first_seen"],
          "last_seen": row["last_seen"],
          "source": "l3_fallback",
      }
  ```

  Step 3 — 在 `api.py` 中添加端点:
  ```python

      caller_scope: str = Depends(require_access_scope),
  ) -> dict[str, Any]:
      result = await audit.skill_usage(
          skill_id=skill_id,
          since=since,
          l2_base_url=l2_url,
      )
      return result
  ```
