## Deviations from Plan

### Adaptation: D9 L2 endpoint URL

**Found during:** Task 5
**Issue:** The planned L2 endpoint `/v1/memory/{skill_id}/usage` does not exist in the current L2 memory-engine codebase (scouted `tools/eaasp-l2-memory-engine/src/` — available endpoints are `/api/v1/memory/search`, `/api/v1/memory/anchors`, `/api/v1/events/ingest`).
**Fix:** Used the planned URL path as the target; L3 gracefully falls through to `source: "l3_fallback"` when L2 returns non-200. The end-to-end behavior is correct — L3 fallback aggregates from its own `telemetry_events` table.
**Files modified:** `tools/eaasp-l3-governance/src/eaasp_l3_governance/audit.py`

### Auto-fix: Pydantic `_tiebreaker` private field issue

**Found during:** Task 8
**Issue:** `_tiebreaker: int = 0` was treated as a Pydantic private attribute (not a regular field), causing all `_tiebreaker` values to silently default to 0 in tests.
**Fix:** Changed to `tiebreaker: int = Field(default=0, alias="_tiebreaker")` pattern per Pydantic v2 conventions. Tests use `_tiebreaker=` alias; code accesses `event.tiebreaker`.
**Files modified:** `tools/eaasp-l3-governance/src/eaasp_l3_governance/audit.py`

### Auto-fix: `asyncio.sleep` removal changes since-filter test semantics

**Found during:** Task 8
**Issue:** After removing `asyncio.sleep(1.1)`, both events in `test_query_since_filter` share the same `received_at` timestamp. The strict `>` filter correctly excludes both events.
**Fix:** Updated test assertion — `since=a.received_at` returns 0 rows (both events have the same timestamp). Tiebreaker-based ordering verified via `ORDER BY tiebreaker DESC`.
**Files modified:** `tools/eaasp-l3-governance/tests/test_audit.py`

### Auto-fix: Python `non-default after default` parameter error

**Found during:** Task 7
**Issue:** `session_id: str = Path(..., ...)` is a "default" parameter in Python's eyes, but `body: SessionValidateRequest` (no default) follows it → `SyntaxError`.
**Fix:** Used `Annotated[str, Path(...)]` pattern which FastAPI supports natively. Added `from typing import Annotated`.
**Files modified:** `tools/eaasp-l3-governance/src/eaasp_l3_governance/api.py`

## Decisions Made

1. **RBAC as Depends() not middleware** — `require_access_scope()` is a reusable FastAPI dependency injected per-endpoint, not a global middleware. This gives per-endpoint control over which routes are scope-guarded.
2. **L2-primary + L3-fallback telemetry** — D9 queries L2's HTTP API first (5s timeout), falls back to L3 local aggregation. When L2 gains a dedicated usage endpoint, the primary path activates automatically.
3. **Wildcard `*` scope semantics** — Callers with `X-Session-Scope: *` bypass all RBAC checks; hooks with `access_scope: *` skip namespace conflict guard. This preserves the wildcard-as-open-door pattern from D11.
4. **Tiebreaker column over clock control** — D26 chose a `tiebreaker INTEGER` column over `freezegun`/`time_machine`. Zero new test dependencies; tests inject explicit tiebreaker values for deterministic ordering.
