<decisions>
## Implementation Decisions

### D74 — EmitEvent gRPC reverse channel (full wire)
- **D-01:** Full implementation — L1 acts as gRPC client to L4 gRPC server. Wire L1→L4 event push for `EventStreamEntry` messages.
- **D-02:** Proto already has `EmitEvent` RPC at `runtime.proto:76` and `EventStreamEntry` at L263. Current runtime stub at `service.rs:597` is L4→L1 (standard). The reverse channel means L1 becomes a gRPC client calling L4's gRPC server.
- **D-03:** L4 FastAPI lifespan starts a gRPC server listening for EmitEvent calls. L1 runtime gets a gRPC client that pushes events when generated (PRE_COMPACT, tool use, etc.).
- **D-04:** ~100-150 LOC. Requires ADR governance gate for contract extension.
- **D-05:** `EmitEvent` contract stays OPTIONAL per ADR-V2-001 — fallback is no-op if L4 not available.

### D130 — Cancel token parent→child propagation (full fix)
- **D-06:** Full implementation — add `ChildCancellationToken.cancel()` propagation semantic + parent→child wiring in AgentExecutor.
- **D-07:** AgentExecutor holds session-lifetime parent token (from `SessionInterruptRegistry`). Each `UserMessage` turn creates `parent.child()` as per-turn token. Cancel parent → child propagates automatically → harness reads cancellation immediately, no channel round-trip needed.
- **D-08:** ~30-50 LOC in `crates/grid-engine/src/agent/harness.rs` + related executor plumbing.

### D106 — MAX_TURNS configurable
- **D-09:** `MAX_TURNS_FOR_BUDGET=50` (hardcoded at `harness.rs:123`) → `AgentLoopConfig` field with configurable default of 50. Follows D102/D3 wiring precedent.
- **D-10:** ~5 LOC — 1 struct field + config wire.

### D139 — 双 Terminate semantics (doc-only)
- **D-11:** Document the two Terminate paths in the contract: (1) normal session end via `TerminateRequest`, (2) timeout/error-imposed termination. Clarify when each applies and enforcement expectations.
- **D-12:** ~20 LOC doc in contract spec + ADR note.

### D90 — WS ToolResult schema
- **D-13:** Add `tool_name: Option<String>` field to `ServerMessage::ToolResult` at `crates/grid-server/`.
- **D-14:** ~5 LOC schema change.

### the agent's Discretion
- D74 ADR structure and gRPC client/server wiring details
- D130 parent→child token pattern (Tokio CancellationToken::child_token or manual)
- Plan structure — D74+D139 (contract) vs D106+D130 (engine) can be parallel
- D74 L4 gRPC server port configuration

</decisions>
