  **The Fix**:
  Replace the snapshot approach with a live-connected token. Two implementation options:

  **Option A (recommended — less invasive):** Create a new `LiveCancellationToken` wrapper type in `cancellation_tree.rs` that holds references to the `SharedFlag` fields and implements an `is_cancelled()` method that checks the live flags:

  pub struct LiveCancellationToken {
      session_flag: SharedFlag,
      turn_flag: SharedFlag,

  impl LiveCancellationToken {
      pub fn is_cancelled(&self) -> bool {
          self.session_flag.is_cancelled() || self.turn_flag.is_cancelled()

  }
