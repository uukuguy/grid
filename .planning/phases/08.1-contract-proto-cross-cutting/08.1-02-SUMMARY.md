## Accomplishments

- D106 (ENGINE-07): `AgentLoopConfig.task_budget_multiplier` replaces hardcoded `MAX_TURNS_FOR_BUDGET=50` — long-running agents can now exceed 50 turns by configuring the multiplier. Threat mitigation: `.max(1)` guard prevents zero-budget DOS.
- D130 (ENGINE-08): `LiveCancelToken` replaces `CancellationToken` in `AgentLoopConfig` — eliminates the per-turn snapshot staleness bug where mid-turn session cancellation was invisible to the harness loop. `session kill` CLI now immediately interrupts in-flight LLM calls.

## Decisions Made

- **LiveCancelToken over child_token pattern:** `CancellationToken::child()` still creates a token with an owned `Arc<AtomicBool>`; the "child" can read the parent's flag but cannot be connected to external shared flags. `LiveCancelToken` directly wraps the `TurnToken`'s `SharedFlag` references, checking them on every `is_cancelled()` call.
- **execute_parallel signature change:** Updated from `&CancellationToken` to `&LiveCancelToken` since it's the only consumer (called from harness). Uses `cancellation.clone()` instead of `cancellation.child()` — equivalent semantics since `LiveCancelToken` is already live-connected.
- **task_budget_multiplier guard:** Added `.max(1)` clamp per T-08.1-05 threat mitigation to prevent zero-budget DOS from a misconfigured multiplier.
