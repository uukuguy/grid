### Q2 — ChunkType envelope JSON shape + ServerMessage 24 变体分类

**Recommended answer (envelope):**
```json
{"type": "chunk", "session_id": "<sid>", "chunk_type": <wire int 1-9>, "payload": {...}}
```
其中 `payload` 字段与对应 proto message JSON 序列化结果同构 (例如 TEXT_DELTA payload = `{"text": "..."}`; TOOL_START payload = `{"tool_id": "...", "tool_name": "...", "input": {...}}`; TOOL_RESULT payload = `{"tool_id": "...", "output": "...", "success": true}`)。

**Classification of 24 ServerMessage variants** (per `ws.rs:40-165`):

| Variant | Class | Wire shape | Wire ChunkType (if chunk) |
|---|---|---|---|
| TextDelta | **chunk** | `{type:"chunk", chunk_type:1, payload:{text}}` | 1 TEXT_DELTA |
| TextComplete | **chunk** | `{type:"chunk", chunk_type:5, payload:{text}}` | 5 DONE (session-level text close — debatable, may be control) |
| ThinkingDelta | **chunk** | `{type:"chunk", chunk_type:2, payload:{text}}` | 2 THINKING |
| ThinkingComplete | **chunk** | (subsume in next chunk) | 8 THINKING_TRACE (end-of-turn summary per 5.3 spec) |
| ToolStart | **chunk** | `{type:"chunk", chunk_type:3, payload:{tool_id, tool_name, input}}` | 3 TOOL_START |
| ToolResult | **chunk** | `{type:"chunk", chunk_type:4, payload:{tool_id, output, success}}` | 4 TOOL_RESULT |
| ToolExecution | **chunk** | (合并入 TOOL_RESULT or 留作 control) | — (legacy; deprecate?) |
| TokenBudgetUpdate | **control** | `{type:"token_budget_update", session_id, budget}` | — (不属于 ChunkType) |
| Typing | **control** | `{type:"typing", session_id, state}` | — |
| Error (model error mid-stream) | **chunk** | `{type:"chunk", chunk_type:6, payload:{message}}` | 6 ERROR |
| Error (session-level) | **control** | `{type:"error", session_id, message}` | — |
| Done | **control** | `{type:"done", session_id}` | — (session-level, not chunk DONE which is per-turn) |
| SessionCreated | **control** | `{type:"session_created", session_id}` | — |
| ContextDegraded | **control** | `{type:"context_degraded", session_id, level, usage_pct}` | — |
| MemoryFlushed | **control** | `{type:"memory_flushed", session_id, facts_count}` | — |
| ApprovalRequired | **control** | `{type:"approval_required", session_id, tool_name, tool_id, risk_level}` | — |

### Q3 — SERVER-04 auth: 模式真相 (重要更正!)

**Recommended answer:** 现状代码 **只有 3 种 AuthMode** (None / ApiKey / Full), 不是 CONTEXT.md/RESEARCH 输入提示的 4 种 (None/ApiKey/Hmac/Hybrid)。HMAC 是 ApiKey 模式内部的 hashing primitive (HMAC-SHA256 hash API key for storage per ADR-003), 不是独立 mode。

**Concrete classification:**
| Mode | Prod-ready | Dev/Test | Audit | 5.4 行动 |
|---|---|---|---|---|
| `None` | ❌ | ✓ (本机 dev) | `warn_if_insecure()` 已打 warn | 保留, 仅文档强化 |
| `ApiKey` (default) | ✓ | ✓ | HMAC-SHA256 hash storage + `GRID_HMAC_SECRET` mandatory env | 主力硬化路径; 加 HMAC signature middleware (per ADR-003 §X-Grid-Signature header) |
| `Full` (JWT) | ✓ (preserve for grid-platform) | — | 已实 `validate_jwt()` 但 server 路径未深度走过 | **5.4 OOS** — grid-platform dormant per ADR-V2-024; 留 stub 但不写测试 |

**SC#4 测试覆盖:**
- `tests/test_auth_modes.rs`:
  - `test_auth_none_allows_all` — mode=None 不检查 header
  - `test_auth_apikey_valid` — 正确 X-API-Key header 200
  - `test_auth_apikey_invalid` — 错误 key 401
  - `test_auth_apikey_expired` — 过期 key 401
  - `test_auth_hmac_signature_valid` — X-Grid-Signature + X-Grid-Timestamp + body 哈希正确 200
  - `test_auth_hmac_signature_replay_window` — timestamp ±5min 之外的请求 401

**ADR-003 relation:** ADR-003 描述 HMAC-SHA256 既用于 API key storage hashing (现有), 又用于 request signature (新增). 后者是 SERVER-04 加固点。

**Rationale:** "4-mode" 是输入提示的误判; 真实 mode 只有 3 个。硬化点在 ApiKey 之内加 signature 层, 而非引入新 mode。

**Evidence:** [VERIFIED] `crates/grid-engine/src/auth/config.rs:37-47` enum AuthMode 三变体

### Q6 — `migrate()` race fix 方案

**Recommended answer:** 改用 `BEGIN EXCLUSIVE TRANSACTION` + 双检 `user_version` 模式。

**关键点:**
- `BEGIN EXCLUSIVE` 在 SQLite 中是文件级别排他锁 (writer lock), 多进程并发 migrate 时只有 1 个进入临界区
- 双检 `user_version`: 进入 txn 后再读一遍, 若已被前一个 winner 升级则直接 COMMIT 退出 (idempotent)
- tokio-rusqlite 用法: `conn.call(|c| { c.execute_batch("BEGIN EXCLUSIVE")?; ... })` (single-fn closure)
- 测试: 用 `std::process::Command::spawn` 启动 2 个子进程同时调 `grid-engine --migrate-test <db_path>` (新加一个隐藏 binary or test harness), 验证只有 1 个 migration log 出现 + db 状态一致

**Regression test 形式:**
```rust
// tests/migrate_race.rs
#[test]
fn test_migrate_no_race_under_concurrent_processes() {
    let db_path = tempdir().path().join("race.db");
    // Pre-create db at user_version=0 (or use migration-script binary)
    let binary = env!("CARGO_BIN_EXE_migrate-tester");
    let handles: Vec<_> = (0..4).map(|_| {
        Command::new(binary).arg(&db_path).spawn().unwrap()
    }).collect();
    for mut h in handles { assert!(h.wait().unwrap().success()); }
    // Assert final user_version == 13 + no "duplicate column" errors in stderr
}
```
需要 `crates/grid-engine/Cargo.toml` `[[bin]] name = "migrate-tester"` 增 1 个 test-only binary。或更简单: 用 `std::thread::spawn` + 不同 `rusqlite::Connection::open` 模拟 (但 thread-shared cache 可能不复现 multi-process race — 建议走 multi-process 路径).

**Rationale:** `BEGIN EXCLUSIVE` 是 SQLite 文档化的并发安全模式; 双检让 idempotent; multi-process spawn 是验证 race 修复的唯一可靠方法 (多线程因 SQLite 有 thread-local 缓存, 不一定复现)。

#[serde(untagged)]
pub enum WsServerMessage {
    Chunk(ChunkEnvelope),
    Control(ControlMessage),
}

#[derive(Serialize)]
pub struct ChunkEnvelope {
    #[serde(rename = "type")] kind: &'static str, // "chunk"
    session_id: String,
    chunk_type: u32,  // wire int 1-9
    payload: serde_json::Value,

pub fn map_event(sid: &str, ev: AgentEvent) -> WsServerMessage {
    match ev {
        AgentEvent::TextDelta{text} => chunk(sid, 1, serde_json::json!({"text": text})),
        AgentEvent::ThinkingDelta{text} => chunk(sid, 2, serde_json::json!({"text": text})),
        AgentEvent::ToolStart{tool_id, tool_name, input} =>
            chunk(sid, 3, serde_json::json!({"tool_id": tool_id, "tool_name": tool_name, "input": input})),
        AgentEvent::ToolResult{tool_id, output, success, ..} =>
            chunk(sid, 4, serde_json::json!({"tool_id": tool_id, "output": output, "success": success})),

        AgentEvent::Done | AgentEvent::Completed(_) => control_done(sid),

        _ => control_passthrough(sid, ev),
    }
}

### 5.4 `migrate()` race fix

```rust
// crates/grid-engine/src/db/mod.rs (replace migrate())
pub fn migrate(conn: &Connection) -> rusqlite::Result<()> {
    // BEGIN EXCLUSIVE acquires writer lock immediately, blocks other writers.
    // Multiple processes serialize through this lock; only one runs migrations.
    conn.execute_batch("BEGIN EXCLUSIVE")?;

    // Double-check: re-read user_version inside the transaction.
    // If a prior winner already migrated, exit clean (idempotent).
    let version: u32 = conn.pragma_query_value(None, "user_version", |row| row.get(0))?;
    if version >= CURRENT_VERSION {
        conn.execute_batch("COMMIT")?;
        return Ok(());
    }

    info!(from = version, to = CURRENT_VERSION, "Running database migration (exclusive)");
    let migrations = get_migrations();
    for migration in migrations {
        if migration.version > version {
            migration.execute(conn)?;
            info!("Applied migration v{}", migration.version);
        }
    }
    conn.pragma_update(None, "user_version", CURRENT_VERSION)?;
    conn.execute_batch("COMMIT")?;
    info!("Migration to v{CURRENT_VERSION} complete");
    Ok(())
}
```

            event_id: format!("stop-{}-{}", self.session_id, chrono::Utc::now().timestamp_millis()),
            session_id: self.session_id.clone(),
            anchor_type: "session_trajectory".into(),
            data_ref: ctx.tool_result.as_ref().map(|v| v.to_string()),
            snapshot_hash: None,
            source_system: Some("grid-server".into()),
        };
        if let Err(e) = self.client.write_anchor(&anchor_req).await {
            warn!(error = %e, "L2 trajectory anchor write failed (non-fatal)");
        }
        // Write file with session summary for FTS5 search
        let file_req = WriteFileRequest {
            memory_id: None,
            scope: format!("session:{}", self.session_id),
            category: "session_trajectory".into(),
            content: format!("Session {} terminated at stop hook", self.session_id),
            evidence_refs: None,
            status: Some("agent_suggested".into()),
        };
        if let Err(e) = self.client.write_file(&file_req).await {
            warn!(error = %e, "L2 trajectory file write failed (non-fatal)");
        }
        Ok(StopHookDecision::Noop) // Never inject; pure observability
    }
}
