<decisions>
## Implementation Decisions

### GA1 — SERVER-02 形态: in-process

- **D-01:** **grid-server 保持 in-process 调 grid-engine**(现有 `AppState.agent_supervisor: Arc<AgentRuntime>` + `agent_handle: AgentExecutorHandle`),**不**新增 grid-runtime subprocess + gRPC client。理由:per ADR-V2-024 双轴模型,grid-server 是 Grid 独立产品 (dormant) 的核心组件;EAASP-via-server 路径是未来推测,5.4 不 pre-invest。新增 gRPC layer 大工作量 + 双进程运维 + approval_gate/interaction_gate 跨进程重设,都不是当前 milestone 的优先发力。
- **D-02:** **ROADMAP SC#2 重写**(由 plan-phase 提交)— 旧文本提到 "grid-server 调 grid-runtime (:50051) 16-method gRPC + `make verify-dual-runtime`" 全部移除;新文本聚焦 "grid-server in-process 调 grid-engine 走完 Initialize/SendResponse/Terminate 三主路径 + chunk relay + hook envelope 透传;session id 在 server-side state 与 engine-side state 一致;集成测试 `crates/grid-server/tests/` 内 ≥3 case PASS"。不提 gRPC,不提 `:50051`,不绑 `make verify-dual-runtime`。

### GA2 — SERVER-01 WS wire alignment + path

- **D-03:** **WS server→client 完全切到 ChunkType-native wire**(对齐 contract-v1.2.0,wire 0-9)。删除/deprecate 现有 `ws.rs:40-180` 14-variant `ServerMessage` enum 的 model-output 相关变体(text_delta / thinking_delta / tool_start / tool_result / token_budget_update / typing 等);改为 thin envelope `{type: "chunk", chunk_type: <wire int>, payload: <proto-shaped json>}`。
- **D-04:** **保留 server-only control messages**(非 ChunkType): `session_created` / `error` / `done` / `approval_required` / `interaction_requested` / `memory_flushed` / `context_degraded` 等 — 这些是 server-protocol 控制信令,不是 model output,不强行塞 ChunkType。Wire shape `{type: "<control_name>", ...payload}`。
- **D-05:** **WS path 切到 `/v1/sessions/{id}/stream` + 保留 `/ws?session_id=` deprecation alias**(复用 5.2 D-03 deprecation 模式):访问 `/ws` 时 server 写一行 stderr / log `warning: '/ws' is deprecated, use '/v1/sessions/{id}/stream' instead (will be removed when no in-tree consumers remain)`,删除时机依 5.2 D-04 grep 归零判据。
- **D-06:** **client→server 不动**(保留现有 `ClientMessage` 4-variant: send_message / cancel / approval_response / interaction_response)。理由:ChunkType 语义是 model output,不适用于 user intent。
- **D-15:** **SC#1 reconnect 缩为 simple reconnect**(new connection succeeds + first SessionCreated frame)。**chunk_id resume semantics defer 到 5.5+**(per RESEARCH §6 R8 + 2026-05-21 plan-checker W1 ratification by user)。理由:chunk_id resume 需 server-side chunk buffer + `?since_chunk_id=` replay, 与 current `AgentEvent` broadcast channel design 不匹, 是 independent feature 适合单独 phase。ROADMAP SC#1 文本同步编辑(由 Task 5.4-01-07 同期负责删除 "从 last chunk_id" 续传 clause + 加 "(chunk_id resume defer 到 5.5+)" 标注)。

### GA3 — SERVER-03 L2 wiring

- **D-07:** **grid-server → L2 memory engine 通过复用现有 `crates/grid-runtime/src/l2_memory_client.rs` 实现**,**不** Rust 端 reimpl FTS5+HNSW(Python L2 是 SSOT),**不**走 "Stop hook 写本地 SQLite + EAASP pull" 松耦合方案。Pros:与 grid-runtime 一致避免双套 schema implementation;L2 仍是 single source of truth;ADR-V2-015 hybrid retrieval 路径不变。
- **D-08:** **L2 client 抽到 `grid-engine` 新 module `grid-engine::l2`**(类比现有 `grid-engine::audit` / `grid-engine::metering` layering),**不**新建独立 `crates/grid-l2-client` crate。Consumer = grid-runtime + grid-server(2 个),独立 crate over-engineering;新 module 与 grid-engine 现有按能力分层一致。
- **D-09:** **Stop hook 写 trajectory 端到端**:grid-server 接收到 Stop hook envelope(per ADR-V2-006)后,调 `grid-engine::l2::write_anchor` / `write_file` 把 trajectory 落到 L2 memory engine;SC#3 ≥2 case 集成测试覆盖 "Stop hook fire → L2 写入成功"。

### GA4+5 — Scope fold-in 决策

- **D-10:** **WATCH-04 + WATCH-07 都 fold 进 5.4**(SC#5 保留原 ROADMAP 文本):D142 + D143 在 grid-runtime/config.rs + claude-code-runtime/service.py 各 ~20 LOC 加 `EAASP_DEPLOYMENT_MODE` 读取 + `max_sessions=1` gate(per ADR-V2-019 per-session 模式);WATCH-07 待 WATCH-04 close 后做 ADR-V2-019 status flip Proposed → Accepted + F1-F4 lint pass。
- **D-11:** **NEW-A2 fold 进 5.4**:`migrate()` 加 `BEGIN EXCLUSIVE` transaction + 双检 `user_version`(读 → 进入 txn → 再读)。SC#3 文本里 "tokio-rusqlite migration 跑通且幂等" 直接对应 NEW-A2 修复后状态。~30-50 LOC + 1 多进程并发 migration regression test。
- **D-12:** **NEW-F3 fold 进 5.4**(代码 + ADR 同 phase):silent-fallback removal 在 `crates/grid-engine/src/providers/{openai,anthropic}.rs` + `crates/grid-engine/src/providers/config.rs` + `grid-server/src/config.rs` 等路径补 strict 校验(LLM_PROVIDER unset → Err 不 default;OPENAI_NO_PROXY 显式 / RUST_LOG fallback always-include grid_* modules)。与 MEMORY.md "No Fallback" feedback always-on rule 对齐。1 ADR 顺手成稿。
- **D-13:** **NEW-A3 / NEW-F4 / grid-cli mod output E0583 / NEW-L1 defer 到 5.5**:domain 与 server 不一致(前三是 CLI / TUI,后一是 Python L2 engine);加进来会裂 phase domain + 增加 plan 数量。5.5 milestone close phase 再统一处理。
- **D-14:** **预估 plan 数 = 2-3**:plan-phase 评估 SERVER-01/02 一组 + SERVER-03+L2 wiring 一组(含 NEW-A2)+ SERVER-04/05+WATCH-04+07+NEW-F3 一组(也可能合 2 plans 取决于工时分布)。

### the agent's Discretion

留给 planner / researcher 决定的 default policy 类问题(GA6/7/8):
