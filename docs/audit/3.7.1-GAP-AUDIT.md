### E-05: Error UX — no `fix:` line, no retryable/permanent, no TTY switch
**File:** `crates/grid-cli/src/main.rs:92-104`
**Snippet (lines 92-103):**
```rust
if let Err(e) = run_command(cli.command, &state).await {
    eprintln!("Error: {}", e);
    // NEW-A3 (Phase 5.5): downcast to typed GridError so variant-specific
    // exit codes (SessionNotFound=4, AuthFailed=3, ...) are preserved.
    let exit_code: ExitCode = e
        .downcast_ref::<GridError>()
        .cloned()
        .map(ExitCode::from)
        .unwrap_or_else(|| GridError::other(e.to_string()).into());
    std::process::exit(exit_code as i32);
}
```
`eprintln!("Error: {}", e)` prints the error chain. Does NOT include `fix: grid doctor --repair` hint. Does NOT distinguish retryable vs permanent. Does NOT detect TTY for text-vs-JSON. `cli.output` is honored only by `output::print_output` for success-path structured output; error path bypasses it entirely.
